## 优化原则

### 1. 工具推荐策略

本模块的特殊性在于：它教的是"怎么搭和用智能体"，所以工具推荐要分两层——搭建平台和辅助工具。

#### 智能体搭建平台

| 类型 | 推荐工具 | 适合场景 |
|------|----------|----------|
| 可视化工作流 | Coze（字节） | 新手首选，拖拽式搭建，免费额度够用 |
| 长期工作台 | OpenClaw | 多渠道接入、后台长期运行 |
| 执行型工作台 | Claude Code、Kiro | 代码/文档协作、项目级任务 |
| 本地模型 | Ollama + 开源模型 | 隐私敏感场景、离线使用 |

#### 辅助工具（遵循 course-optimization-guide.md）

| 用途 | 推荐工具 |
|------|----------|
| 记录日志和状态 | 飞书多维表格（首选）、Notion Database |
| 存模板和 SOP | 飞书文档、语雀 |
| 消息通知 | 飞书机器人、企业微信机器人 |
| 流程串联 | Coze（可视化工作流）、飞书多维表格（自动提醒） |

### 2. 工具推荐的表述方式

- 每次提到具体平台时标注"截至 2026 年 3 月，功能和价格可能变化，请以官网为准"
- OpenClaw 和 Claude Code 作为课程标题中的代表性产品，重点讲用法
- 同时给出替代方案，避免绑定单一产品
- 新手阶段优先推荐 Coze（门槛最低），进阶再推荐 OpenClaw / Claude Code

### 3. 课间衔接

- 第 1 课结尾 → "知道什么是智能体了，下节课看看长期稳定的任务怎么交给工作台"
- 第 2 课结尾 → "长期工作台搞定了，下节课看看变化快、上下文重的任务怎么用执行型工作台"
- 第 3 课结尾 → "两种工作台都试过了，最后一课来定自动化的边界：哪些该自动、哪些必须人工确认"
- 第 4 课结尾 → "智能体的边界定好了，接下来带着这套方法去看具体场景 → 超级个体场景库"

### 4. 与前后模块的衔接

#### 与第二模块（大模型实战库）的衔接
- 第二模块最后一课（model-eval-sheet）结尾应该提到"评测表建好后，带着你的默认模型去学智能体"
- 第三模块第 1 课开头应该接上："上个模块你选好了模型，现在来解决下一个问题：怎么让模型不只是聊天，而是真正帮你执行任务"

#### 与第四模块（场景库）的衔接
- 第三模块最后一课结尾 → 引导去场景库
- 场景库的课程可以引用智能体模块的权限分层和回退机制

## 逐课优化要点

### 第 1 课：agent-basics

- 补工具：用 Coze 搭"每日信息摘要助手"的具体步骤（截图级别不需要，但要写清在哪里操作）
- 补案例：小王，运营岗，之前用聊天框手动整理日报 → 用 Coze 搭了摘要助手 → 每天省 30 分钟
- 补翻车：① 把聊天框当智能体，任务断了没人知道 ② 一上来就做全自动外发，结果发错内容
- 判断表加使用说明：拿到一个任务后，花 5 分钟填这张表，3 个以上"是"才值得做

### 第 2 课：openclaw-style

- 补工具：OpenClaw 的基本概念 + Coze 作为替代方案的对比
- 补案例：小李，自媒体运营，用长期工作台做每日素材收集 → 权限只开到 L2 → 稳定跑了 2 周后才开 L3
- 补翻车：① 一上来就开写入权限，自动覆盖了正式文档 ② 没设日志，出问题后不知道哪一步错了
- 落地顺序加时间标注

### 第 3 课：claude-code-style

- 补工具：Claude Code 的安装和基本用法 + Kiro 作为替代方案
- 补案例：小张，前端开发，用执行型工作台修复一个跨文件 Bug → 用任务描述模板 → 一次通过
- 补翻车：① 目标没写清楚，模型把整个模块重构了 ② 没写"不做什么"，模型引入了 3 个新依赖
- 任务描述模板加一个填写示例

### 第 4 课：automation-boundary

- 补工具：日志记录用飞书多维表格，停止开关用 Coze 的条件节点
- 补案例：小赵，电商客服主管，从手动回复 → 半自动草稿 → 可控自动回复的升级过程
- 补翻车：① 没设停止开关，自动化跑了一晚上发了 200 条重复消息 ② 没有人工接管点，流程卡住只能重来
- 健康流程示例展开，加上每一步用什么工具
