# 课程内容优化指南

本指南总结了"AI 超级个体入门"模块优化过程中形成的标准，供后续模块优化时参考。

## 目标用户画像

- 中国用户，25-35 岁上班族
- AI 使用经验浅，可能只用过聊天
- 需要具体操作步骤，不需要抽象概念
- 时间有限，每节课 15-30 分钟

## 每节课必须包含的结构

### 1. 开头：这节课你要拿到什么
- 3 个具体的交付物（不是"理解XX概念"，而是"拿到XX模板"）

### 2. 正文：跟着做的操作步骤
- 每个步骤标明预计时间
- 涉及工具时给出具体产品名（面向中国用户优先推荐国产工具）
- 每个步骤有明确的完成标准（✅ 完成标准：xxx）

### 3. 案例：一个完整的用户故事
- 给案例人物一个名字（小王、小李、小张）
- 写清楚他的职业和场景
- 用表格展示他的操作过程和效果变化
- 引用他的一句感受总结

### 4. 常见翻车/常见问题
- 用 ❌/✅ 对比格式
- 每个翻车场景写清：错误做法 → 为什么错 → 正确做法

### 5. 自检清单
- 5 条左右的 checkbox 式检查项
- 全部打勾 = 本课完成

### 6. 本课落地动作
- 3 件今天就能做的事
- 与下节课衔接

## 工具推荐原则

### 对话助手
- 优先推荐：豆包、元宝（腾讯）、通义千问（阿里）、Kimi
- 进阶推荐：Claude、ChatGPT

### 信息核查（不叫"搜索验证"）
- 推荐：秘塔 AI 搜索、Perplexity
- 必须强调：AI 搜索是"线索入口"，关键数据（价格、版本、政策、日期）必须去官网确认
- 不要把 AI 搜索等同于"最新事实来源"

### 知识沉淀
- 推荐：飞书文档、Notion、语雀

### 流程串联（不叫"自动化"或"自动搬运"）
- 推荐：飞书多维表格（自动提醒、状态流转）、Coze（可视化工作流）
- 新手阶段强调用工具自带功能，不要一上来就接 API
- 不推荐 Zapier（中国用户不常用），用 Coze 替代

## 写作风格

### 要做的
- 用生活类比解释抽象概念（"像一个随叫随到的实习生"）
- 用表格展示对比信息
- 给出可直接复制的模板和提示词
- 课与课之间有明确衔接（"上节课写的模板"、"下节课会用到"）
- 精简表达，说清楚就行，不要反复强调同一个观点

### 不要做的
- 不要只讲概念不给操作步骤
- 不要推荐工具时只说品类不说具体产品名
- 不要用"微信提醒""手机闹钟"充当自动化工具
- 不要在一节课里重复 3 遍以上同一个观点
- 不要用"最好的""最强的"等主观评价

## 修改涉及的文件

每节课的内容分布在 3 个位置：

1. `src/content/lessons/{module-id}/{slug}.md` — 课程正文（Markdown）
2. `src/content/modules/{module}.ts` — 课程元数据（title、content、details、fullContent）
3. `src/components/lesson-page/LessonVisualGuide.tsx` — 可视化引导组件（如果有的话）

修改时三个位置要保持一致。

## 质量检查清单

优化完一节课后，确认：

- [ ] markdown 正文有"跟着做"的操作步骤
- [ ] 涉及工具时给了具体产品名
- [ ] 有至少 1 个用户案例（带名字、职业、操作记录）
- [ ] 有 ❌/✅ 对比的翻车场景
- [ ] 有自检清单
- [ ] 有本课落地动作
- [ ] fullContent 与 markdown 内容一致
- [ ] details 与新内容一致
- [ ] TypeScript 编译通过
- [ ] AI 搜索工具没有被描述为"最新事实来源"
- [ ] 自动化工具的描述准确，不牵强
