# 大模型实战库 — 模块优化指南

## 模块概况

大模型实战库是第二大模块（id: `llm`），共 4 节课：

| # | slug | 标题 | 核心主题 |
|---|------|------|----------|
| 1 | model-framework | 选模型先看五个维度 | 选型框架 |
| 2 | model-landscape | 主流模型怎么分工 | 模型地图 |
| 3 | api-governance | API 接入与权限管理 | 治理规则 |
| 4 | model-eval-sheet | 建立你自己的模型评测表 | 评测落地 |

涉及文件：
- `src/content/lessons/llm/*.md` — 课程正文
- `src/content/modules/llm.ts` — 课程元数据
- `src/content/moduleEnhancements/llm.ts` — 模型对比数据面板

## 当前内容诊断

### 整体问题

对照 course-optimization-guide.md 的标准，4 节课普遍存在以下差距：

| 优化标准 | 当前状态 |
|----------|----------|
| 开头 3 个具体交付物 | ✅ 已有，但部分偏抽象（"一套框架"不如"一张填好的表"） |
| 操作步骤标明预计时间 | ❌ 缺失，没有时间标注 |
| 工具推荐给具体产品名 | ❌ 大量"模型 A / 模型 B"占位符，没有具体推荐 |
| 完整用户案例（带名字、职业、表格） | ❌ 全部缺失 |
| ❌/✅ 对比的翻车场景 | ❌ 全部缺失 |
| 自检清单（checkbox） | ❌ 全部缺失 |
| 本课落地动作（3 件事） | ⚠️ 有"完成标准"但不是"今天就能做的动作" |
| 课与课之间的衔接 | ❌ 缺失，4 节课各自独立 |

### 逐课问题

#### 第 1 课：model-framework（选型框架）

- 框架讲得清楚，但缺少"跟着做"的分步操作
- 评测表用"模型 A / 模型 B"，应该给出真实模型名（结合 moduleEnhancements 里的数据）
- 缺案例：需要一个"小王用这个框架选出默认模型"的完整故事
- 缺翻车场景：比如"只看排行榜切模型导致成本翻倍"
- 试跑提示词模板很好，保留

#### 第 2 课：model-landscape（模型地图）

- "不要背名字"的理念好，但全文没给一个具体模型名，读者看完还是不知道选什么
- 应该结合 moduleEnhancements 里的 6 个模型数据，给出具体的任务-模型映射建议
- 模型地图表格全是"模型 A / 模型 B"占位符，需要填入真实推荐
- 缺案例和翻车场景
- "长文本不要只看上下文窗口"这段很好，保留并加操作步骤

#### 第 3 课：api-governance（API 治理）

- 结构最完整的一课，4 步流程清晰
- 但缺少具体工具推荐（用什么记录？飞书多维表格？Notion？）
- Key 分离讲了原则但没给操作步骤（去哪里创建 Key？怎么命名？）
- 缺案例：需要一个"小李第一次接 API 踩坑"的故事
- 缺翻车场景：比如"Key 写进公开仓库被盗刷"

#### 第 4 课：model-eval-sheet（评测表）

- 和第 1 课有较多内容重叠（都在讲评测表）
- 需要明确差异：第 1 课是"快速选型"，第 4 课是"系统评测"
- 10 个样本的建议很好，但缺少具体的执行时间和步骤拆分
- 缺案例和翻车场景

## 优化原则

### 1. 模型推荐策略

本模块的特殊性在于：它教的就是"怎么选模型"，所以不能像第一模块那样直接说"用豆包"。

策略：
- 给出具体模型名作为"示例"，但始终强调"用你自己的评测结果决定"
- 利用 moduleEnhancements 里已有的 6 个模型数据（GPT-5.4 系列、Claude 系列、DeepSeek V3.2）
- 中文场景额外提及：豆包（字节）、通义千问（阿里）、Kimi（月之暗面）
- 每次提到模型时标注"截至 2026 年 3 月，版本和价格可能变化，请以官网为准"

### 2. 工具推荐（遵循 course-optimization-guide.md）

| 用途 | 推荐工具 |
|------|----------|
| 记录调用和评测 | 飞书多维表格（首选）、Notion Database |
| 存模板和 SOP | 飞书文档、语雀 |
| API 测试 | 各平台官方 Playground、Coze |
| 可视化工作流 | Coze（替代 Zapier） |

### 3. 课间衔接

- 第 1 课结尾 → "下节课我们用这个框架，看看主流模型怎么分工"
- 第 2 课结尾 → "模型地图画好了，下节课学怎么安全地接入 API"
- 第 3 课结尾 → "API 接好了，最后一课用系统评测把选型落地"
- 第 4 课结尾 → "评测表建好后，带着你的默认模型去学智能体 →"

### 4. 与第一模块的衔接

第一模块最后一课（first-week-plan）提到"拿到这些再去学模型选型和智能体"。
第二模块第 1 课开头应该接上："上个模块你已经有了模板和工作流，现在来解决一个问题：模板里用的模型，到底选对了没有？"
