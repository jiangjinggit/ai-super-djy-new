## 这节课你要拿到什么

- 一张 5 维任务适配评分表
- 3 类最适合新手起手的任务
- 3 类现阶段不建议碰的任务
- 一份 OpenClaw / Claude Code / Cowork / Coze 选型速查

## 为什么要先判断任务适配性

很多人不是不会用智能体，而是把不适合的任务硬往智能体上套。

结果通常是：

- 花很多时间搭流程
- 最后还是要手动补救
- 对智能体能力形成错误期待

先判断任务，再选工具，效率会高很多。

## 5 维评分表

给你手头的一件任务按下面 5 项打分，每项 `0` 或 `1`：

| 维度 | 判断问题 | 得分 |
|------|---------|------|
| 输入来源 | 输入是不是相对固定，不需要你每次重新解释？ | 0 / 1 |
| 完成标准 | 什么算完成，能不能写成规则？ | 0 / 1 |
| 任务频率 | 这件事是不是每周至少会重复 3 次？ | 0 / 1 |
| 风险控制 | 出错后是不是容易发现、容易回退？ | 0 / 1 |
| 工具支撑 | 现有工具链能不能接上这个流程？ | 0 / 1 |

评分解读：

- `5 分`：强烈建议做
- `3-4 分`：先做半自动
- `1-2 分`：先别做，继续手动更划算

## 最适合起手的 3 类任务

| 任务类型 | 为什么适合 | 推荐工具 |
|---------|-----------|---------|
| 信息汇总 | 输入稳定，输出通常是摘要或表格 | OpenClaw / Coze |
| 内容整理 | 规则容易写清，人工校验也快 | Cowork / Coze |
| 代码辅助 | 范围可控，验证方式明确 | Claude Code |

这三类任务共同特点：

- 结果容易看出对不对
- 失败成本不高
- 容易迭代成模板

## 新手阶段不要碰的 3 类任务

| 任务类型 | 为什么不适合 |
|---------|-------------|
| 直接对外承诺的任务 | 一旦出错，代价高，无法轻易回退 |
| 高度依赖最新事实的任务 | 容易被过期信息或错误引用带偏 |
| 规则自己都说不清的任务 | 任务定义都没稳定，做成智能体只会放大混乱 |

## 工具选型速查

| 工具 | 最适合什么 | 不适合什么 |
|------|-----------|-----------|
| OpenClaw | 多渠道信息收集、主动提醒、定时任务 | 临时一次性探索 |
| Claude Code | 本地代码、文档、命令行任务 | 长期后台监控 |
| Cowork | 文件夹资料整理、研究汇总、成品文档草稿 | 仓库级代码修改、云端长期后台任务 |
| Coze | 快速搭低门槛流程和 Bot | 高度复杂的项目级协作 |

## Claude Code 和 Cowork 最容易混淆的地方

它们都能“做事”，但不是同一种“做事”。

| 判断问题 | 选 Claude Code | 选 Cowork |
|---------|---------------|----------|
| 你面对的是仓库、脚本、终端吗？ | 是 | 否 |
| 你面对的是文件夹、资料、研究材料吗？ | 否 | 是 |
| 你需要看 diff、跑测试、改代码吗？ | 是 | 否 |
| 你需要整理文档、表格、演示文稿吗？ | 否 | 是 |

先把这一步分清，后面学 Claude 模块时才不会把两条线混在一起。

## 跟着做：给 3 个任务打分（⏱ 10 分钟）

从你最近的工作里选 3 件最想自动化的事，比如：

- 每周整理行业信息
- 修复重复出现的小 Bug
- 整理会议纪要

每件事都跑一遍 5 维评分表，然后只选分数最高的那一件继续推进。

不要同时开 3 条线。新手最容易输在贪多。

## 📋 案例：小王的 3 个候选任务

| 任务 | 得分 | 结论 |
|------|------|------|
| 每日行业资讯汇总 | 5 分 | 立刻做，适合 OpenClaw |
| 修一个老项目的偶发 Bug | 3 分 | 可做，但先小范围用 Claude Code |
| 整理一堆研究资料做汇报 | 4 分 | 值得做，先用 Cowork 跑一版成品草稿 |
| 自动帮客户回复报价邮件 | 1 分 | 暂时不做，风险太高 |

关键不是“哪个最酷”，而是“哪个最容易先做出稳定正反馈”。

## 常见翻车

### 翻车 1：频率太低的任务也想自动化

一件事一个月才做一次，先手动通常更省时间。

### 翻车 2：把“高价值”误以为“适合做”

越高价值的任务，往往越应该先半自动，而不是直接交给系统全权处理。

### 翻车 3：工具选型靠感觉

“这个工具最近很火”不是选型理由。输入结构、执行方式、确认机制，才是关键。

### 翻车 4：把 Cowork 和 Claude Code 混着学

一个更适合仓库与终端，一个更适合桌面文件和资料。起步阶段同时混着学，通常只会增加理解负担。

## ✅ 本课落地动作

- 用评分表给 3 个真实任务打分
- 只留下分数最高的 1 个任务
- 根据任务特点，先决定它更适合 OpenClaw、Claude Code、Cowork 还是 Coze
