## 从一个真实问题开始

很多人会说：“我每天都在用 ChatGPT，那我不就在用智能体吗？”

大多数情况下，不是。

你可能在用一个很强的对话工具，但它未必能：

- 读外部环境
- 自己拆步骤
- 真正调用工具执行动作
- 记住任务状态，下次接着做

能聊天，不等于能持续完成任务。

## 先把术语说清楚

在这套课里：

- `智能体` = `Agent`
- `AI 智能体` 和 `Agent` 指的是同一类东西，只是中文和英文叫法不同

但有两个常见混淆词也要一起分开：

- `Agentic workflow`：带一点代理式流程感的自动化，不一定是完整 Agent
- `聊天工具`：能对话，但不一定能感知、行动和记忆

所以更准确的说法是：

不是所有“像智能体”的产品，都是完整 Agent。

## 智能体的 4 个核心能力

| 能力 | 它具体做什么 | 少了会怎样 |
|------|-------------|-----------|
| 感知 | 读文件、网页、数据库、消息、日历 | 只能等你把信息手动贴给它 |
| 规划 | 把大任务拆成步骤，决定先后顺序 | 容易想到哪做到哪，无法稳定推进 |
| 行动 | 调工具、写文件、发消息、执行命令 | 只能“建议你怎么做”，无法真正动手 |
| 记忆 | 记录状态、偏好、历史决策 | 每次都像第一次见面，重复问同样的问题 |

速记方法：

`感知` 是看见世界，`规划` 是想清顺序，`行动` 是真的去做，`记忆` 是下次还能续上。

## 对比：智能体 vs 普通 AI 对话

| 维度 | 普通 AI 对话 | 智能体 |
|------|-------------|-------|
| 输入来源 | 主要靠你手动提供 | 能主动读环境或接收触发 |
| 工具调用 | 有时能建议 | 能真正调用工具执行 |
| 任务延续 | 一轮问答结束就断 | 可跨多步、多轮持续推进 |
| 出错处理 | 你发现、你纠正 | 需要规则、日志、确认点来控制 |
| 适合场景 | 临时问答、草稿生成 | 重复任务、流程闭环、持续协作 |

## 3 个最常见误区

### 误区 1：把聊天框当智能体

如果一个工具不会读环境、不会调工具、没有状态记录，它再聪明也还是聊天框。

很多产品会说自己“有 agent 能力”或“支持 agentic workflow”，这并不自动等于它就是完整智能体。

### 误区 2：智能体 = 全自动

真正成熟的做法不是“一键全自动”，而是知道哪些步骤能自动、哪些步骤必须停下来等你确认。

### 误区 3：现在的智能体已经很可靠

智能体已经很强，但还远远没到“随便放手”的程度。权限、日志、回退和人工确认点，依然是必需品。

## 跟着做：4 问判断卡（⏱ 5 分钟）

拿一个你常用的 AI 工具，连续问自己这 4 个问题：

1. 它能主动读取我授权的外部环境吗？
2. 它能把目标拆成步骤并说明顺序吗？
3. 它能调用工具把事情真正做完吗？
4. 它能记住这次任务状态，下次继续吗？

判断标准：

- `4 个是`：可以算完整智能体
- `2-3 个是`：半自动助手
- `0-1 个是`：主要还是对话工具

## 📋 案例：同样是“做周报”，差别在哪

| 做法 | 过程 | 结果 |
|------|------|------|
| 聊天框做周报 | 你手动贴任务清单，它生成一版文案 | 快，但每次都要重新喂上下文 |
| 智能体做周报 | 它读取任务列表、整理结构、生成草稿、记录反馈 | 前期配置更重，后期更稳、更省时间 |

关键区别不是“谁更会写”，而是谁能把完整流程接起来。

## 别把所有“Claude 能做事的产品”看成一种

这一步很重要，因为很多人会在这里混淆：

- `Claude Code CLI`：偏仓库、代码、终端、工程任务
- `Cowork`：偏文件、资料、桌面端、成品文档
- `OpenClaw`：偏长期在线、主动提醒、定时任务

它们都可能带有“智能体感”，但入口、工作对象和最适合的任务完全不同。

一句话速记：

- `要动仓库`，先想 CLI
- `要动文件和资料`，先想 Cowork
- `要长期主动跑`，先想 OpenClaw

## 常见翻车

### 翻车 1：把“能联网”误认为“能感知”

能联网只说明它可能能查资料，不等于它能读你自己的项目、日历、任务清单。

### 翻车 2：把“会写代码”误认为“会行动”

它会输出命令，不等于它会真的执行，更不等于执行过程是可控的。

### 翻车 3：把“记住聊天记录”误认为“有记忆系统”

真正的记忆不是把聊天堆长，而是把关键信息写入正确的位置，下次还能被稳定读取。

### 翻车 4：把“能做代理”误以为“都是同一种代理”

Cowork 和 Claude Code 都很强，但一个更像桌面工作台，一个更像终端执行台。入口选错了，学习体验会差很多。

- 用 4 问判断卡评估你当前在用的 2 个 AI 工具
- 把它们分别归类为：聊天工具 / 半自动助手 / 真正智能体
- 再补一句判断：你现在更需要的是 OpenClaw、Claude Code CLI，还是 Cowork
- 顺手记住一条术语口径：智能体就是 Agent，但 Agentic workflow 不一定是完整 Agent
