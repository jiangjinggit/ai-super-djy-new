## 这节课你要拿到什么

- 一张"这个任务适不适合做智能体"的判断表（填完就能用）

- 一份可直接复制的智能体任务说明模板

## 真正的智能体至少要有 5 个组件

很多产品只是把聊天框换个名字，不算智能体。最小可用智能体至少包括：

| 组件 | 说明 | 举例 |
|------|------|------|
| 任务输入 | 触发智能体开始工作的信号 | 定时触发、新消息、新文件 |
| 上下文读取 | 读取完成任务所需的背景信息 | 读网页、读文档、查数据库 |
| 工具调用 | 调用外部能力完成具体动作 | 搜索、生成文本、发通知 |
| 结果输出 | 交付可用的成果 | 一份摘要、一条消息、一个文件 |
| 状态记录 | 记住做到哪一步了 | 日志、标记、进度表 |

少任何一个，稳定性都会大打折扣。

> 💡 简单判断：如果一个"智能体"既不能调工具、也不能记录状态、更不能推动任务前进，那它本质上还是个聊天框。

## 跟着做：用 Coze 搭一个"每日信息摘要助手"

不要一上来做全自动运营。先做下面这种低风险任务。

> 截至 2026 年 3 月，Coze（扣子）是字节跳动的可视化智能体搭建平台，免费额度够新手练手。功能和价格可能变化，请以官网为准。

### 第 1 步：明确任务目标（⏱ 5 分钟）

写清楚这个智能体要做什么：

```text
任务：每天把 3 个信息源的内容整理成一份摘要
输入：新闻链接、收藏夹文章、群聊讨论记录
输出：一份 3-5 条要点的摘要 + 一份待办列表
频率：每天一次
```

✅ 完成标准：能用一句话说清楚"它要做什么、输入什么、输出什么"

### 第 2 步：在 Coze 里创建智能体（⏱ 10 分钟）

1. 打开 [coze.cn](https://www.coze.cn)，注册并登录
2. 点击"创建 Bot"
3. 填写名称："每日信息摘要助手"
4. 在"人设与回复逻辑"里粘贴下面的任务说明（见后文模板）
5. 在"插件"里添加"网页读取"插件（用于读取链接内容）

✅ 完成标准：Bot 创建成功，能在对话框里和它对话

### 第 3 步：测试并调整（⏱ 10 分钟）

1. 给它发一条新闻链接，看它能不能提取要点
2. 给它发一段群聊记录，看它能不能整理成摘要
3. 如果输出格式不对，回去调整"人设与回复逻辑"里的指令

✅ 完成标准：发 3 条不同类型的输入，输出格式一致、要点准确

### 第 4 步：记录结果（⏱ 5 分钟）

把每次的输入和输出记录到飞书多维表格或 Notion：

| 日期 | 输入来源 | 摘要要点数 | 是否需要人工修改 | 备注 |
|------|----------|-----------|----------------|------|
| 3/31 | 新闻链接 x3 | 5 条 | 否 | 格式稳定 |

✅ 完成标准：有一张记录表，能看到每次执行的效果

## 可直接复制的智能体任务说明模板

```text
你的角色：每日信息摘要助手

目标：
每天把输入内容整理成一份摘要，供人工查看和决策。

执行步骤：
1. 阅读全部输入
2. 删除噪音信息（广告、重复、无关内容）
3. 提取 3-5 条核心结论
4. 标记待确认项（数据未核实、来源不明的内容）
5. 按固定格式输出

输出格式：
## 今日摘要
- 要点 1：...
- 要点 2：...
- 要点 3：...

## 待确认
- [ ] ...

## 待办建议
- [ ] ...

禁止：
- 不要编造事实
- 不要对外发送内容
- 信息不足时必须标记"待确认"
```

## 用这张表判断任务是否适合做智能体（⏱ 5 分钟）

拿到一个任务后，花 5 分钟填这张表。3 个以上答"是"，才值得做成智能体：

| 问题 | 是/否 | 你的情况 |
|------|-------|---------|
| 输入来源是否明确？ | | |
| 完成标准是否能写成规则？ | | |
| 是否需要调用工具（搜索、读取、生成）？ | | |
| 能否记录执行状态（做到哪一步了）？ | | |
| 失败时能否安全停止（不会造成损失）？ | | |

> 如果只有 1-2 个"是"，说明这个任务更适合用普通工作流或手动完成。

## 适合起手的 3 种智能体任务

| 任务 | 为什么适合 | 推荐工具 |
|------|-----------|---------|
| 每日报告摘要 | 输入固定、输出格式稳定、低风险 | Coze、飞书多维表格 |
| 常见问题分拣 | 规则明确、可以先生成草稿再人工确认 | Coze、飞书机器人 |
| 内容素材收集与归档 | 高频重复、只读操作、不涉及外发 | Coze、Notion |

不适合起手的任务：

| 任务 | 为什么不适合 |
|------|-------------|
| 自动对外回复 | 出错成本高，需要先跑通半自动流程 |
| 自动下单或支付 | 涉及资金，必须有完善的确认和回退机制 |
| 自动改生产数据 | 不可逆操作，新手阶段风险太大 |

## 📋 案例：小王用 Coze 搭了第一个智能体

小王，互联网公司运营岗，每天要花 40 分钟从 5 个渠道收集行业动态，手动整理成日报发给团队。

### 他的操作过程

| 步骤 | 之前（手动） | 之后（智能体辅助） |
|------|-------------|-------------------|
| 收集信息 | 逐个打开 5 个网站，复制粘贴 | Coze Bot 自动读取链接内容 |
| 整理要点 | 自己读完再写摘要 | Bot 生成 3-5 条要点草稿 |
| 标记待办 | 凭记忆列待办 | Bot 自动标记"待确认"项 |
| 输出日报 | 手动排版发飞书 | 人工确认后复制到飞书文档 |
| 每日耗时 | 40 分钟 | 15 分钟（含人工确认） |

> "最大的变化不是省时间，而是不会漏信息了。以前靠记忆，现在有固定流程。" —— 小王

### 小王踩过的坑

他一开始想直接让 Bot 自动发日报到群里，结果第二天发现 Bot 把一条未核实的消息也发出去了。后来改成"Bot 生成草稿 → 自己确认 → 手动发送"，反而更稳定。

## 常见翻车

### 翻车 1：把聊天框当智能体用

❌ 错误做法：每天手动打开 ChatGPT / 豆包，把内容粘贴进去让它整理，觉得"这就是智能体了"

为什么错：没有固定输入源、没有状态记录、没有失败处理。明天换个格式就得重新教它。

✅ 正确做法：用 Coze 搭一个 Bot，配好输入插件和输出格式，每次执行流程一致，结果可追踪。

### 翻车 2：一上来就做全自动外发

❌ 错误做法：第一天就让智能体自动把生成的内容发到客户群 / 公开渠道

为什么错：没有经过测试期，输出质量不稳定，一旦发错内容，影响远大于手动操作。

✅ 正确做法：前两周只让智能体生成草稿，自己确认后再手动发送。确认稳定后再考虑半自动外发。

### 翻车 3：不记录执行结果

❌ 错误做法：智能体跑完就完了，不看输出质量，也不记录。

为什么错：无法判断智能体是否在退化（模型更新、输入变化都可能导致输出质量下降）。

✅ 正确做法：用飞书多维表格记录每次执行的输入、输出和是否需要人工修改，每周回顾一次。
