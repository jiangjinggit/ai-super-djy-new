## 这节课你要拿到什么

- 一个真正“能跑起来”的最小智能体定义
- 一个适合新手落地的智能体闭环示例
- 一套判断“这是不是智能体”的检查标准

## 真正的智能体至少要有 5 个组件

很多产品只是把聊天框换个名字，不算智能体。最小可用智能体至少包括：

1. `任务输入`
2. `上下文读取`
3. `工具调用`
4. `结果输出`
5. `状态记录`

少任何一个，稳定性都会大打折扣。

## 先做一个最简单的智能体：每日信息摘要助手

不要一上来做全自动运营。先做下面这种低风险任务：

### 目标

每天固定时间，把几个信息源整理成一份摘要。

### 输入

- 新闻链接
- 收藏夹文章
- 群聊讨论记录

### 处理流程

1. 读取原始内容
2. 去掉无关信息
3. 提炼 3-5 条要点
4. 标出风险和待确认项
5. 存到固定位置

### 输出

- 一份摘要
- 一份待办列表

## 用这张表判断任务是否适合做智能体

| 问题 | 是/否 |
| --- | --- |
| 输入来源是否明确 |  |
| 完成标准是否明确 |  |
| 是否需要调工具 |  |
| 能否记录状态 |  |
| 失败时能否停止 |  |

只有大多数都能答“是”，这个任务才值得做成智能体。

## 智能体和普通工作流的区别

普通工作流更像固定流水线，智能体则会在流程中做选择，例如：

- 先搜索还是先读取知识库
- 信息不足时是否继续
- 结果不完整时是否回头补资料

但新手阶段不要追求太多“自主决策”，先把流程打稳。

## 适合起手的 3 种智能体任务

- 每日报告摘要
- 常见问题分拣
- 内容素材收集与归档

不适合起手的任务：

- 自动对外回复
- 自动下单或支付
- 自动改生产数据

## 可直接复制的智能体任务说明

```text
你的角色：每日信息摘要助手

目标：
每天把输入内容整理成一份摘要，供人工查看和决策。

执行步骤：
1. 阅读全部输入
2. 删除噪音信息
3. 提取 3-5 条核心结论
4. 标记待确认项
5. 输出到固定格式

禁止：
- 不要编造事实
- 不要对外发送内容
- 信息不足时必须标记“待确认”
```

## 这节课的完成标准

你做完后，至少应该有：

- 1 个适合做智能体的任务
- 1 张任务适配判断表
- 1 份最小执行说明
- 1 套失败停止条件

这才是“开始做智能体”，不是只看产品演示。
