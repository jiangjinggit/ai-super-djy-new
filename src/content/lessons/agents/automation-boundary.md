## 这节课你要拿到什么

- 一种判断“该不该自动化”的方法
- 一份必须人工确认的动作清单
- 一套回退和止损机制

## 不要先追求全自动，先做正确分工

任何流程都先分成 3 层：

- `自动执行层`：信息收集、格式整理、草稿生成
- `人工确认层`：判断、审批、外发
- `禁止自动层`：高危命令、支付、批量覆盖

这样你做自动化时，风险边界会非常清楚。

## 哪些动作必须人工确认

下面这些动作默认都要确认：

- 对外发送消息
- 改生产数据
- 删除或覆盖文件
- 下单、支付、扣费
- 批量执行不可逆动作

任何一个想绕过人工确认，都要有充分理由和回退机制。

## 自动化适配评分表

给一个流程打分，满足越多越适合自动化：

| 维度 | 说明 | 是否满足 |
| --- | --- | --- |
| 高频 | 一周至少发生 3 次 |  |
| 规则明确 | 完成标准清楚 |  |
| 输入稳定 | 数据来源固定 |  |
| 风险可控 | 出错可回退 |  |
| 可观察 | 有日志和结果可看 |  |

如果只满足 1-2 项，不建议自动化。

## 回退机制最少要有这 3 个

1. `停止开关`
2. `日志记录`
3. `人工接管点`

### 停止开关

一旦发现输出异常，能立刻停止，不继续放大错误。

### 日志记录

至少知道：

- 什么时候执行
- 做了什么
- 输出到哪里
- 谁确认过

### 人工接管点

流程卡住时，人可以无缝接手，不用重来。

## 一个健康的自动化流程示例

1. 收集原始资料
2. AI 生成草稿
3. 人工确认
4. 写入正式结果
5. 记录日志

不要把第 3 步删掉。

## 这节课的完成标准

你做完后，至少应该有：

- 1 张自动化适配评分表
- 1 份必须人工确认的动作清单
- 1 套停止和回退机制

做到这一步，自动化才算“能用且可控”。
