## 学习目标

- 只用国内 6 家模型做代码选型：`GLM / Kimi / MiniMax / 豆包 / DeepSeek / Qwen`
- 比较代码模型时，把价格一起算进去，不再只看 benchmark
- 给代码任务配出 `复杂主力 / 日常默认 / 低成本跑量 / 回退模型`

## 核心知识

### 1. 先定角色，再选模型

先别问“谁最强”，先问“这个任务要什么角色”。

| 角色 | 典型任务 | 最该看的指标 |
| --- | --- | --- |
| 复杂主力 | 跨文件改造、关键功能、复杂 Debug | 理解深度、稳定性、一次通过率 |
| 日常默认 | 常规功能、小改动、补测试 | 速度、价格、整体均衡 |
| 低成本跑量 | 批量分析、预处理、初稿 | 输入/输出单价、吞吐 |
| 回退模型 | 主力限流、风格不合、中文协作偏多 | 切换成本、可得性、中文体验 |

| 家族 | 代表模型 | 官方价格 | 更适合什么任务 | 课程判断 |

### 3. 比价时，别只看输入价

1. **代码任务要看输出价。** 真正写代码、补测试、生成解释时，输出 tokens 往往不少，很多模型是“输入便宜，输出不便宜”。
2. **要看长上下文阶梯价。** `Qwen` 和 `豆包` 都有明显的上下文分档；你一旦经常读大仓库，账单会变。
3. **要看套餐还是按量。** `MiniMax` 这点最明显：官网同时有 `Token Plan / Coding Plan / Pay as you go`，不要把不同计费口径硬比到一起。

### 4. 课程里的最小推荐

这是课程判断，不是厂商原话；依据是 2026 年 4 月 3 日官网的模型定位和公开价格。

- **国内默认模型：** `Qwen3-Coder-Flash`
- **复杂主力：** `Kimi K2` 或 `GLM-5 / GLM-5-Code`
- **低成本跑量：** `DeepSeek-V3.2`
- **中文前端回退：** `Doubao-Seed-Code`
- **已经买了 MiniMax 套餐再考虑：** `MiniMax-M2.7 / M2.5`

## 作业

拿你自己的真实仓库，选 3 类任务各跑一次：

1. 一个小改动
2. 一个中等复杂功能
3. 一个跨文件高风险改动

每类任务只记录 4 个结果：

- 一次通过率
- 返工时间
- 总 token 成本
- 你愿不愿意把它设成默认模型
