## 学习目标

- 用 OpenClaw 搭一个飞书 FAQ 客服机器人：自动回答常见问题，复杂问题升级到人工
- 掌握飞书机器人接入配置、FAQ 知识库管理、升级条件设置的完整流程
- 扩展方案：对接 FastGPT 知识库，处理更大量的文档型 FAQ

## 适合谁

- 需要在飞书群里处理大量重复问题的团队（客服、HR、运营）
- 电商卖家想把旺旺/飞书群的常见咨询自动化的人
- 已完成 OpenClaw 实战模块，想搭第一个实用客服场景的人

## 前置条件

- OpenClaw 已部署运行
- 飞书应用已创建，机器人可以收发消息（参考 OpenClaw 模块的飞书接入步骤）
- 已整理出你的高频 FAQ（至少 20 个问题及标准答案）

## 完整操作步骤

### Step 1：整理 FAQ 知识库（最关键的一步）

在开始配置 AI 之前，先把你的 FAQ 整理好。这是影响效果最大的因素。

**FAQ 整理模板**：

从你过去 1-3 个月的客服记录/飞书消息里，整理出频率最高的 30-50 个问题，按类别分组：

```markdown
## FAQ 知识库

### 物流类
Q: 什么时候发货？
A: 一般 1-2 个工作日内发货，特殊节假日会顺延，具体时间参考订单状态通知。

Q: 物流单号怎么查？
A: 下单成功后会短信通知，也可在[系统名称]里查看订单详情页。

### 产品/服务类
Q: [具体问题]
A: [标准回答，包含必要的细节]

### 退款/售后类
Q: 如何申请退款？
A: 联系客服（@[具体人名]）或通过[渠道]提交退款申请，处理时间 3-5 个工作日。
```

**关键提示**：
- A（答案）要写完整，不要留"视情况而定"这类模糊表述
- 需要人工判断的问题，答案写"请@[负责人]处理"而不是 AI 自己猜

---

### Step 2：配置 SOUL.md（客服角色）

```markdown
## 客服助手规则

### 角色定位
你是[公司/团队名称]的客服助手，负责回答飞书群中的常见问题。

### 工作范围
- 可以回答：FAQ 知识库中列出的标准问题
- 必须升级：涉及退款、投诉、重大问题、账户安全
- 必须回复但不能擅自承诺：涉及价格优惠、特殊处理

### 回复风格
- 语气：友好、简洁、专业
- 长度：一般不超过 100 字，有必要时可以更长
- 格式：直接给答案，不要说"根据我的知识库"这类前缀
- 末尾：需要时可以加"还有其他问题吗？"

### 绝对禁止
- 不允许做任何价格承诺或优惠承诺
- 不允许处理投诉，必须转人工
- 不允许讨论竞品
- 不允许透露内部流程细节
```

---

### Step 3：配置 AGENTS.md（消息路由规则）

```markdown
## 客服消息路由规则

### 自动回复条件
以下问题类型直接回复，不需要确认：
- 物流查询类（发货时间、物流状态）
- 基础产品介绍类
- 使用教程类
- 活动规则类

### 立即升级条件（@[负责人飞书账号]）
以下情况立即停止自动回复，@相关人员处理：
- 消息含关键词：退款、投诉、差评、骗、质量问题、不满意、太贵、要举报
- 同一用户 24 小时内发了 3 条以上未解决的消息
- 问题涉及金额超过 [填写你的阈值] 元
- 看起来是新客户或大客户的首次咨询

### 非工作时间处理（22:00-8:00 及周末）
- 自动回复："您好，您的消息已收到，将在工作时间（周一到周五 9:00-18:00）内回复，如有紧急情况请拨打 [电话]"
- 同时记录到 daily notes，供工作时间跟进

### 未匹配问题处理
超出 FAQ 知识库范围的问题：
- 回复："您的问题我还不能直接回答，我帮您转接相关同事处理"
- 自动 @[负责人] 并附上原始问题
```

---

### Step 4：把 FAQ 知识库接入 OpenClaw

**方案 A：直接写入 SOUL.md（适合 50 条以内的 FAQ）**

把整理好的 FAQ 直接加到 SOUL.md 的末尾：

```markdown
## FAQ 知识库

[把你的 FAQ 列表粘贴在这里]
```

OpenClaw 每次回复时会参考这些内容。

**方案 B：对接 FastGPT（适合大量文档或复杂 FAQ）**

1. 在 FastGPT（cloud.fastgpt.in）创建知识库，上传 FAQ 文档
2. 获取 FastGPT API Key 和 API URL
3. 在 OpenClaw 里安装 HTTP skill（用于调用外部 API）：

```bash
openclaw env set FASTGPT_API_KEY "你的FastGPT API Key"
openclaw env set FASTGPT_API_URL "https://api.fastgpt.in/api/v1/chat/completions"
```

4. 在 AGENTS.md 里添加 FastGPT 调用规则：

```markdown
### FastGPT 知识库调用
对于超出简单 FAQ 范围的产品/服务问题，先查询 FastGPT 知识库（API 配置见环境变量），
如果找到相关答案则直接回复，如果找不到则升级到人工。
```

### Step 5：飞书机器人接入配置（如果还没配）

如果飞书机器人还没有接入 OpenClaw，按以下步骤：

1. 登录飞书开放平台（open.feishu.cn）→ 进入你的自建应用
2. 能力 → 机器人 → 开启
3. 事件订阅 → 添加 `im.message.receive_v1` 事件
4. 权限管理 → 添加以下权限：
   - `im:message`（读取消息）
   - `im:message:send_as_bot`（发送消息）
   - `im:chat`（读取群信息）
5. 把机器人拉进你的客服群或频道

验证：

```bash
openclaw gateway status
# 应显示 feishu: connected
```

在飞书群里 @机器人 发"你好"，确认能收到回复。

### Step 6：测试和验收

**测试脚本（准备 10 个测试消息）**：

1. 一个标准 FAQ 问题 → 期待：直接回答，无需升级
2. 一个 FAQ 范围边缘的问题 → 期待：回答或请求更多信息
3. 退款相关消息 → 期待：立即 @ 负责人，不自动回答
4. 投诉类消息 → 期待：立即 @ 负责人
5. 非工作时间测试 → 把本地时间调成 22:30，看是否触发非工作时间回复
6. 完全超出范围的问题 → 期待：转人工，而不是乱编答案
7. 同一用户发 4 条未解决消息 → 期待：升级
8. 一个 @ 机器人但不是问题的消息（如"谢谢"）→ 期待：简短友好回复
9. 一个价格问题 → 期待：按标准说法回复，不承诺折扣
10. 一个需要查询订单的问题 → 期待：引导到自助查询渠道

## 常见问题与调试

**问题 1：机器人回答了不该自动回答的问题（如退款投诉）**

原因：升级关键词没有覆盖到。

解决：把触发误判的消息加到升级关键词列表里。

**问题 2：机器人答案不准确，编了一个不存在的答案**

原因：FAQ 知识库里没有这条内容，但机器人没有触发"不知道就升级"的规则。

解决：在 SOUL.md 里加强兜底规则：
```markdown
对于 FAQ 知识库中没有的问题，必须回复"这个问题需要由我们的专员为您解答"，
然后升级到人工，不允许猜测或自行发挥。
```

**问题 3：非工作时间有人发消息，白天忘记跟进**

解决：在 AGENTS.md 里加一条 Cron 任务，每天早上 9:00 把昨晚未处理的消息列表推送给负责人。

**问题 4：飞书群里 @ 了机器人但没有回应**

检查步骤：
```bash
openclaw logs --follow
# 在飞书发消息，看日志里是否有收到消息记录
```
如果日志没有记录：检查飞书事件订阅是否已启用。
