## 学习目标

- 用 OpenClaw 搭一套行业与竞品监控日报系统：每天自动抓取 → AI 过滤整理 → 飞书推送
- 掌握多信息源配置、三层分流（立即推送 / 日报 / 静默）和过滤规则的写法
- 扩展到竞品价格监控、关键词告警等场景

## 适合谁

- 需要每天关注行业动态、竞品动向的产品经理、运营、创业者
- 想从"手动刷行业信息"转变为"AI 主动推送精选内容"的人
- 已完成 OpenClaw 实战模块的用户

## 前置条件

- OpenClaw 已部署运行，飞书已接入
- 有明确的监控方向（行业/竞品/关键词）
- 国内模型（DeepSeek / Kimi / 豆包）已配置好

---

## 主工具路径

| 环节 | 工具 | 说明 |
|------|------|------|
| 定时触发 | OpenClaw Cron | 工作日早 8:30 自动执行 |
| 内容抓取 | web-search skill | 访问指定信息源 |
| AI 过滤整理 | DeepSeek / Kimi | 按规则筛选 + 生成摘要 |
| 推送渠道 | 飞书频道 / 飞书群 | 日报消息推送 |
| 紧急告警 | 飞书 @ 个人 | 重要事件立即通知 |

## 完整操作步骤

### Step 1：确定你的监控清单

在开始配置前，先写下来你要监控什么，分两类：

**A. 行业信息源**（每天浏览，看宏观趋势）

| 类别 | 推荐来源 | URL |
|------|---------|-----|
| 科技热点 | 36氪 | https://36kr.com/information/technology/ |
| AI 工具 | 少数派 | https://sspai.com/tag/AI |
| 行业深度 | 虎嗅 | https://www.huxiu.com/channel/105.html |
| 开发者 | GitHub Trending | https://github.com/trending |
| 产品/创业 | PingWest | https://www.pingwest.com/ |

**B. 竞品/关键词监控**（出现变化才通知）

列出 3-5 个竞品的官网或特定页面，以及你最关心的关键词（如：融资、新产品、降价）。

---

### Step 2：安装 web-search skill

```bash
openclaw skill install web-search
# 验证
openclaw skill list | grep web-search
```

测试：在飞书里 @机器人 发"搜索 36氪 今日 AI 新闻"，看是否返回结果。

---

### Step 3：配置 AGENTS.md（监控规则）

打开 `~/.openclaw/AGENTS.md`，添加日报规则：

```markdown
## 行业监控日报规则

### 信息源清单
每次执行时按顺序访问以下来源：

**行业信息（每日必看）**
- 36氪科技频道：https://36kr.com/information/technology/
- 少数派 AI 标签：https://sspai.com/tag/AI
- 虎嗅前沿科技：https://www.huxiu.com/channel/105.html
- GitHub Trending（今日）：https://github.com/trending

**竞品监控（有变化才记录）**
- [填写竞品1官网或产品页 URL]
- [填写竞品2更新日志页面 URL]

### 过滤规则

**保留的内容（进入日报）**：
- 发布时间：48 小时内
- 内容类型：产品发布、功能更新、融资新闻、行业报告
- 与我关注的方向相关：[填写你的垂类，如 AI 编程工具/电商 SaaS/内容创作]

**过滤掉的内容（不进日报）**：
- 广告和软文
- 纯转载（来源非原创）
- 与垂类无关的内容
- 昨日已推送的内容

**紧急告警条件（立即推送，@我本人）**：
- 竞品宣布融资超过 1 亿元
- 竞品发布重大版本更新
- 行业出现重大政策变化

### 输出格式

每日日报格式（发到"行业日报"飞书频道）：
```
📰 [日期] 行业日报

1. [标题] — [一句话摘要] [来源链接]
2. [标题] — [一句话摘要] [来源链接]
（最多 5 条）

⚠️ 竞品动态（如有）：[竞品名] [具体变化]
```

紧急告警格式（@我本人）：
```
🚨 紧急：[事件标题]
来源：[URL]
要点：[2-3 句话说明为什么重要]
```
```

### Step 4：配置 Cron 定时任务

编辑 `~/.openclaw/openclaw.json`：

```json
{
  "cron": [
    {
      "pattern": "30 8 * * 1-5",
      "prompt": "执行行业日报：按 AGENTS.md 配置，访问信息源、过滤内容、生成今日日报，推送到飞书「行业日报」频道。如有触发紧急告警条件，立即 @ 我本人。",
      "timezone": "Asia/Shanghai"
    }
  ]
}
```

同时建议加一个 Heartbeat，用于竞品页面的实时监控：

```json
{
  "heartbeat": {
    "interval": 7200,
    "prompt": "快速检查竞品监控列表（见 AGENTS.md），判断是否有满足紧急告警条件的变化。有则立即通知，无则静默。"
  }
}
```

`interval: 7200` = 每 2 小时检查一次。

---

### Step 5：飞书频道配置

建议在飞书里创建两个专用频道：

1. **行业日报**频道：每天接收日报推送（@所有人或特定成员）
2. **竞品告警**频道：紧急事件 @相关人员

在 AGENTS.md 的通知规则里指定：
```markdown
### 通知路由
- 日报推送：飞书「行业日报」频道
- 紧急告警：飞书「竞品告警」频道 + @[你的飞书用户名]
- 普通信息：静默记录到 daily notes，不主动推送
```

---

### Step 6：验证与调优

**第一次手动测试**：

```bash
openclaw run --prompt "执行一次行业日报测试：按 AGENTS.md 配置抓取信息，生成测试日报推送飞书" --once
```

**看效果时检查**：
- 每条内容是否都在 48 小时内？
- 和你的垂类相关度如何？
- 摘要是否简洁（一句话）？
- 有没有推送超过 5 条（太多了要加严过滤）？

**调优方向**：
- 内容太泛：在 AGENTS.md 的过滤规则里加更具体的垂类描述
- 内容太少：放宽时间窗口（48h → 72h）或添加更多信息源
- 摘要太长：在输出格式里注明"每条摘要不超过 30 字"

## 关键配置模板

### 竞品价格监控（进阶）

如果想监控竞品的定价页面是否有变化，在 SOUL.md 加：

```markdown
## 竞品价格监控
监控以下页面的价格信息：
- [竞品1] 定价页：[URL]
- [竞品2] 定价页：[URL]

检测逻辑：对比当前价格与 MEMORY.md 中记录的上次价格，有变化则记入 MEMORY.md 并告警。
```

### Cron 时间参考

| 场景 | 表达式 | 说明 |
|------|--------|------|
| 工作日早报 | `30 8 * * 1-5` | 周一到周五 8:30 |
| 每日晚报 | `0 18 * * 1-5` | 工作日 18:00 |
| 每周行业综述 | `0 9 * * 1` | 每周一 9:00 |
| 每日全天 | `30 8 * * *` | 包含周末 |

## 常见问题与调试

**问题 1：日报内容和我的垂类关联度低**

原因：信息源太广，或垂类描述不够具体。

解决：在 AGENTS.md 的保留规则里加更具体的描述，例如：
```
相关领域：AI 编程工具（不含教育类、消费类 AI）、开发者工具、SaaS
```

**问题 2：每天推送超过 10 条，刷屏了**

解决：在输出格式里明确写"最多 5 条，超过则只保留最重要的"。

**问题 3：Heartbeat 一直没有竞品告警**

正常现象：竞品确实没有更新。可以手动检查：
```bash
openclaw run --prompt "检查竞品告警列表，返回每个竞品的最新状态" --once
```

**问题 4：日报时间不对（比如 16:30 才推送）**

原因：缺少 `timezone: Asia/Shanghai`。加上后重启 OpenClaw：
```bash
openclaw daemon restart
```
