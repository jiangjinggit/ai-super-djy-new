## 学习目标

- 用 FastGPT 搭一个企业内部知识库机器人，接入飞书，让员工可以"问文档"
- 掌握文档整理、FastGPT 知识库配置、飞书机器人对接的完整步骤
- 知道知识库质量的判断标准和常见的优化方法

## 适合谁

- 公司内部有大量文档（制度、FAQ、产品手册、操作指南），但员工找不到、找到了也看不完
- 想搭一个"新人入职问答"或"内部 FAQ 机器人"的团队负责人
- 想用 OpenClaw + FastGPT 组合实现比简单 FAQ 更强的知识库系统的人

## 前置条件

- 有内部文档（Word / PDF / Markdown 均可），至少 5 份以上
- 飞书已接入 OpenClaw（机器人可以收发消息）
- 有一台电脑可以访问 FastGPT 的国内版本（cloud.fastgpt.in）

## 完整操作步骤

### Step 1：整理文档（最关键，经常被跳过）

**原则**：文档质量决定 AI 回答质量。在上传到 FastGPT 之前，先做一次文档清理。

**清理检查表**：
- [ ] 删除过期的版本（如果有"2022 年版"和"2024 年版"，只保留最新的）
- [ ] 处理扫描件（扫描的 PDF 需要先 OCR 提取文字，FastGPT 无法直接读图片）
- [ ] 拆分过长的文档（超过 200 页的建议按章节拆成多个文件）
- [ ] 统一格式（优先用 Word 或 Markdown，PDF 也可以但效果略差）

**文档分类建议**：

```
📁 内部知识库/
  ├── 制度政策/（考勤、报销、入离职流程）
  ├── 产品手册/（产品介绍、功能说明）
  ├── 操作指南/（系统操作步骤）
  ├── 常见问题/（手动整理的 FAQ）
  └── 项目资料/（如果需要的话）
```

---

### Step 2：注册并配置 FastGPT

**注册 FastGPT**：
1. 访问 cloud.fastgpt.in（国内访问，无需代理）
2. 注册账号（手机号注册）
3. 新用户有免费额度，够初期测试使用

**创建知识库**：
1. 左侧菜单 → 知识库 → 新建知识库
2. 命名（如"[公司名] 内部知识库"）
3. 选择向量模型：默认即可（OpenAI ada-002 或阿里云 text-embedding）

**上传文档**：
1. 进入知识库 → 导入数据
2. 选择"文件导入"，上传你整理好的文档（支持 PDF、Word、Markdown、TXT）
3. 分块设置：默认配置一般够用；如果文档很长，可以把"每段最大长度"改为 500-800 字

**等待处理**：FastGPT 会自动把文档切分成段落，做向量化处理，通常 1-5 分钟完成。

---

### Step 3：创建问答应用

1. 左侧菜单 → 应用 → 新建应用 → 选"简易应用"
2. 配置：
   - 名称：[公司名] 内部助手
   - 对话模型：推荐 DeepSeek-V3（便宜且效果好）或 Kimi
   - 知识库：选择你刚创建的知识库

3. 调整系统提示词（关键配置）：

```
你是[公司名]的内部知识助手。

你的职责：
- 根据知识库中的文档，回答员工关于公司制度、产品、操作流程的问题
- 如果知识库中有相关信息，给出准确答案并注明来源文件名
- 如果知识库中没有相关信息，明确告知"我目前的知识库没有这方面的信息，建议联系 [HR/产品/相关负责人]"

限制：
- 不允许猜测或编造不在知识库中的内容
- 不要回答与公司业务无关的问题
- 涉及薪资、人事调动等敏感问题，一律回复"请直接联系 HR"
```

4. 保存配置，点击"对话测试"，用 20 个真实问题验证回答质量。

---

### Step 4：测试并优化（用 20 个真实问题测试）

**测试方法**：从员工实际会问的问题里选 20 个，在 FastGPT 的对话测试界面逐一测试。

**评分标准**：

| 结果 | 处理方式 |
|------|---------|
| 回答正确（✅） | 记录，继续下一个问题 |
| 回答不完整（⚠️） | 找到对应文档段落，补充信息后重新上传 |
| 回答错误（❌） | 找到错误来源，修正文档 |
| 说"不知道"但应该知道（❓） | 检查文档是否上传，或分块是否太细 |

**目标**：20 个问题里，准确回答 ≥ 16 个（80%）才上线。

**优化方法**：

1. 回答质量差的问题 → 在原始文档里把这个问题的答案写得更清晰、更明确
2. 经常回答"不知道"但文档里有 → 检查文档是否已向量化（刷新知识库）
3. 答案太长看不下去 → 在系统提示词里加"回答尽量简洁，控制在 200 字以内"

---

### Step 5：获取 API 并接入 OpenClaw

1. FastGPT 左侧 → 账号 → API 密钥 → 新建 API Key，复制保存
2. 获取应用的 API：在你的应用里 → 外部接口使用 → 复制 API URL

在 OpenClaw 服务器上配置：

```bash
openclaw env set FASTGPT_API_KEY "你的FastGPT API Key"
openclaw env set FASTGPT_APP_URL "https://api.fastgpt.in/api/v1/chat/completions"
```

**在 AGENTS.md 里添加知识库调用规则**：

```markdown
## 内部知识库问答规则

### 触发条件
飞书群中 @机器人 的消息，且问题类型属于：
- 公司制度类（请假、报销、考勤）
- 产品功能类
- 操作流程类
- 新人入职常见问题

### 处理流程
1. 把用户问题发给 FastGPT API（环境变量 FASTGPT_APP_URL）
2. 把 FastGPT 返回的答案直接回复到飞书
3. 如果 FastGPT 说"不知道"，回复：
   "这个问题超出了我的知识范围，请联系相关负责人：[具体说明找谁]"

### 不处理的问题类型
- 涉及个人薪资：直接说"请联系 HR"，不调用知识库
- 投诉建议：转到专用渠道 [飞书群名]
```

---

### Step 6：上线与持续维护

**上线前检查**：
- [ ] 在飞书群里 @ 机器人问 5 个真实问题，确认回答质量
- [ ] 确认"不知道"的回复是准确的（而不是乱编）
- [ ] 确认敏感问题的处理（薪资、人事）不会让 AI 乱回答

**上线后维护**（每月一次）：
1. 查看飞书机器人被问过的问题记录
2. 找出回答不好的问题 → 修正对应文档
3. 如果公司制度或产品有更新 → 更新知识库文档并重新上传
4. 删除过期文档

## 关键配置模板

### FastGPT 系统提示词模板（可直接复制）

```
你是[公司名]内部的智能问答助手。

回答规则：
1. 只根据知识库中的内容回答，回答时注明来源文件名
2. 知识库没有相关内容时，明确说"我没有这方面的信息"
3. 涉及以下敏感话题，一律转人工：薪资、绩效、人事变动、法律纠纷
4. 回答简洁，不超过 300 字，有必要时可以附上相关文档链接

语气：专业、友好，像一个熟悉公司业务的老员工
```

### OpenClaw AGENTS.md 知识库调用片段

```markdown
## 知识库 API 调用
当需要查询内部知识库时，发送 POST 请求到 ${FASTGPT_APP_URL}：
- Authorization: Bearer ${FASTGPT_API_KEY}
- Content-Type: application/json
- Body: {"chatId": "unique-id", "stream": false, "messages": [{"role": "user", "content": "用户问题"}]}

返回的 choices[0].message.content 即为回答。
```

## 常见问题

**问题 1：FastGPT 的回答经常出错或乱编**

原因：原始文档里有不一致的表述，或某些内容写得太模糊。

解决：找到出错的问题，到原始文档里把对应内容改得更明确，然后删除旧的知识库数据并重新上传。

**问题 2：上传了文档但 FastGPT 说不知道**

可能原因：
- 文档还在处理中（等几分钟）
- 文档是扫描的图片 PDF（需要先 OCR）
- 问题的描述方式和文档里差别太大

**问题 3：回答太长，员工不想看**

在系统提示词里加：
```
回答规则：
- 第一行：一句话结论
- 以下：如有必要，提供 2-3 条补充说明
- 全文不超过 150 字
```

**问题 4：飞书里 @ 机器人但调用的是 OpenClaw 的普通对话，不是知识库**

解决：在 AGENTS.md 里明确写触发知识库的条件，让 OpenClaw 知道什么时候应该调用 FastGPT API 而不是直接用 AI 回答。
