## 学习目标

- 用 Cursor 或 Claude Code，把一个具体想法做成可访问的产品并上线
- 掌握产品描述文档的写法、与 AI 协作的迭代节奏、国内部署和收款方案
- 知道 Vibe Coding 能解决什么问题，哪些坑是工具解决不了的

## 适合谁

- 有产品想法但没有开发背景，或有基础但以前没有独立交付过完整产品的人
- 想用 AI 编程工具验证一个小想法的运营、产品、设计、咨询等非技术岗
- 已完成 AI 编程模块，想做第一个真实产品的人

## 前置条件

- 有一个明确的产品想法（哪怕很粗糙）
- 安装了 Cursor（推荐）或能访问 Claude Code
- 会用命令行的基本操作（cd、ls、运行 npm install 之类）
- 对"第一版能用就行，不追求完美"有心理准备

## 完整操作步骤

### Step 1：写产品描述文档（30 分钟）

在开始写代码之前，把想法约束成一张文档。这是最能避免浪费时间的一步。

**模板**：

```
产品名称：[一个简短的名字]

解决的问题：
[谁] 在 [什么情况下] 遇到 [什么具体问题]
（例：内容创作者每周写文章，在选题时不知道什么话题受欢迎）

目标用户：
[具体描述一个典型用户]（不要写"所有人"）

第一版只做这些功能：
1. [功能 1]
2. [功能 2]（最多 3 个，再多就删掉）

第一版不做：
- [明确排除的功能]（写这个很重要，防止功能蔓延）

成功标准：
[具体描述如何判断第一版做完了，例：用户能完成一次完整的 XX 流程]
```

**检验产品描述好不好的标准**：
把这个文档发给一个不了解你想法的朋友，他能不能理解你要做什么。如果不能，说明还需要再具体。

---

### Step 2：用 AI 扮演用户，挖出你没想到的问题

把产品描述文档贴给 Claude 或 Cursor，用这个 Prompt：

```
你是我的目标用户：[具体描述用户]。

我要做这个产品：[粘贴你的产品描述文档]

请扮演这个用户，提出 5 个你会有的疑问或疑虑。
不要给我正面鼓励，我需要真实的挑战和潜在的问题。
```

根据 AI 的问题，在开始写代码之前调整你的产品设计。

---

### Step 3：选工具

**国内无代理环境（不能访问 Claude/OpenAI）**：
- 百度秒哒（mida.baidu.com）：零基础，国内，无代码生成网页应用
- Coze（coze.cn）：搭对话型产品，流程类工具

**国内有代理环境（推荐）**：
- Cursor：最成熟，代码质量最稳定，从免费版开始
- Claude Code：适合任务明确后批量执行（需要代理）

**模型选择**（Cursor 里选）：
- Claude Sonnet 4.5 / Claude Opus 4.6：复杂功能首选
- DeepSeek-V3（本地 API Key 接入）：成本便宜，可以用于大量重复任务

---

### Step 4：开始构建，控制范围

**正确的开发节奏**：每次只做一个可以独立运行的功能，做完验证后再加下一个。

**第一条消息怎么写**：

```
我要做一个 [产品名称]，第一步只需要：[单一功能描述]

技术要求：
- 前端：React（或 HTML+CSS，越简单越好）
- 数据存储：本地浏览器 localStorage（第一版不需要数据库）
- 不需要用户登录（第一版）

请帮我生成这个最简单的版本，能运行就行，暂时不用考虑样式好看。
```

**每次迭代的节奏**：
1. 描述一个功能 → AI 生成代码
2. 运行测试，看是否符合预期
3. 有问题：复制完整错误信息给 AI，让它分析修复
4. 符合预期：提交一个 git commit，再做下一个功能

**遇到错误的正确处理方式**：

```
我运行了你的代码，出现了以下错误：
[复制完整错误信息，包括文件名和行号]

我做了什么操作触发了这个错误：[描述操作步骤]

请分析错误原因并给出修复方案。
```

### Step 5：部署上线

**前端部署（静态网站）**：

最快方案 - Vercel：
```bash
# 安装 Vercel CLI
npm install -g vercel

# 在项目目录里执行
vercel

# 按提示操作，几分钟就能拿到一个 https://xxx.vercel.app 地址
```

国内访问慢时的替代方案：
- 阿里云 OSS + CDN（需要备案，但国内访问速度快）
- 腾讯云 COS + CDN（同上）

**有后端的项目（API 服务）**：

快速部署 - 腾讯云函数（国内，无需备案，按调用量计费，便宜）：
```bash
# 安装 Serverless 工具
npm install -g serverless

# 腾讯云登录
serverless login

# 部署
serverless deploy
```

境外替代 - Railway（有免费额度，部署最简单）：
1. railway.app → New Project → Deploy from GitHub
2. 连接你的 GitHub 仓库
3. 配置环境变量
4. 自动部署，一键搞定

### Step 6：收款方案（上线前必须想清楚）

这是最常被忽略的一步，等产品做好了才发现收不了款。

| 方案 | 适合场景 | 费率 | 注意 |
|------|---------|------|------|
| 爱发电 | 知识付费、订阅、买断 | 8% 手续费 | 个人可用，最快开通 |
| 小报童 | 内容订阅、newsletter | 5-10% | 微信生态内传播好 |
| 微信支付（官方） | 正式商业产品 | 0.6% | 需要营业执照 |
| 支付宝（官方） | 同上 | 0.6% | 需要营业执照 |
| Stripe | 面向海外用户 | 2.9%+0.3刀 | 需要境外主体或代理 |
| Lemon Squeezy | 面向海外，含税务处理 | 5%+0.5刀 | 比 Stripe 对个人更友好 |

**推荐路径**：
- 第一个产品：用爱发电，今天就能开通
- 有稳定用户后：注册个体工商户，接微信支付/支付宝
- 目标是海外用户：注册香港或美国实体，用 Stripe

## 关键配置模板

### 用 Claude Code 完成整个项目的 CLAUDE.md（项目规范文件）

在项目根目录创建 `CLAUDE.md`：

```markdown
# 项目说明

## 产品描述
[粘贴你的产品描述文档]

## 技术栈
- 前端：React + TypeScript
- 后端：[如果有的话]
- 数据库：[如果有的话]

## 开发规范
- 每次只改一个功能，不要同时改多个地方
- 改动前先告诉我你要做什么，再动手
- 有不确定的地方先问我，不要猜测我的意图

## 不要做
- 不要添加我没有要求的功能
- 不要重构已经运行正常的代码
- 不要在没有我确认的情况下删除文件
```

## 常见问题

**问题 1：AI 生成的代码运行不起来，不知道怎么办**

把完整的错误信息复制给 AI（Terminal 里的红色文字），不要只说"运行不了"。

**问题 2：做了一大堆，但用户反馈完全不是想要的**

原因：太晚让真实用户看到。第一版主流程做好后就找 3-5 个人测试，别等到"完善"了再发。

**问题 3：部署后产品很慢，国内用户访问几乎打不开**

Vercel 对国内访问慢是正常现象。解决：
- 域名绑定阿里云 CDN（把静态资源缓存到国内节点）
- 或迁移到阿里云/腾讯云部署

**问题 4：做完了没人用**

这不是工具的问题。分发渠道（你在哪个平台有受众）和产品方向选择，比代码本身更重要。先想清楚"谁会为此付钱/使用它"，再开始写代码。
