## 这节课你要拿到什么

- 一张 Claude Code CLI 与 Cowork 的分工表
- 一条 CLI 与 Cowork 的首次启动路径
- 两个低风险验证任务
- 一份常见安装问题排查表

## 先说清两条入口的定位

Claude 系执行工作台现在至少有两条常见入口：

- `Claude Code CLI`
- `Cowork`

它们不是谁替代谁，而是适合不同任务。

| 工具 | 更适合什么 |
|------|-----------|
| Claude Code CLI | 仓库、代码、文档、命令行、diff、脚本 |
| Cowork | 本地文件夹、研究资料、表格、演示文稿、浏览器和连接器 |

## 什么时候优先学 CLI

如果你接下来最想做的是：

- 读项目结构
- 改小范围代码
- 跑命令
- 用 `CLAUDE.md` 管项目规范

那就先学 CLI。

## 什么时候优先看 Cowork

如果你接下来最想做的是：

- 整理一堆文件
- 汇总研究资料
- 生成表格、报告或演示文稿
- 不想打开终端

那就先看 Cowork。

## 安装前提

至少先确认三件事：

- CLI：终端环境、登录方式、目标项目目录
- Cowork：桌面端应用、文件夹权限、目标 Project 或工作目录
- Cowork 如果要跑长任务或定时任务，还需要你的电脑保持唤醒，桌面端保持开启

具体安装和可用平台可能更新，始终以官方文档为准。

## 跟着做：首次启动（⏱ 10 分钟）

### 路线 A：Claude Code CLI

1. 按官方当前推荐方式安装 Claude Code
2. 完成登录或鉴权
3. 进入一个你熟悉的项目目录
4. 启动 Claude Code

### 路线 B：Cowork

1. 打开 Claude Desktop
2. 切换到 Cowork
3. 创建或选择一个 Project / 文件夹
4. 让它读取资料并完成一个小任务

注意：

- Cowork 是桌面端能力，不是网页聊天模式
- 它更适合付费计划和持续任务场景
- 任务执行期间不要随手把桌面端关掉

目标都一样：不是立刻做大任务，而是先验证“它能正确理解当前工作空间”。

## 两个验证任务

### 任务 1：CLI 验证

让它：

- 主要目录是什么
- 关键入口文件在哪
- 这个项目大概在做什么

### 任务 2：Cowork 验证

给它一个真实资料文件夹，让它：

- 找出最相关的文件
- 提炼要点
- 生成一份草稿结果

你要观察的都是同一件事：

- 它读文件是否准确
- 分析是否具体
- 有没有凭空猜测

## 订阅和计费

不管你用哪种接入方式，都应该先设好额度意识：

- 不要默认无限用
- 对高消耗任务保持警惕
- 一开始优先用小任务验证，而不是拿大仓库硬跑
- Cowork 的长任务和多步骤任务通常比普通对话更耗额度

## 常见安装问题

| 问题 | 可能原因 |
|------|---------|
| CLI 启动失败 | 安装没完成或环境不兼容 |
| Cowork 不可用 | 平台、版本或桌面端条件不满足 |
| 读不到项目或文件 | 工作目录、Project 或权限不对 |
| Cowork 中途停掉 | 电脑睡眠、桌面端被关闭、权限被撤销 |
| 响应异常慢 | 网络、环境或当前任务过重 |

- 跑通对应的一次最小验证任务
