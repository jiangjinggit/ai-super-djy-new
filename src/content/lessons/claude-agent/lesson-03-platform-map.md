## 学习目标

- 知道 CLI、IDE、Desktop、Web、Cowork 五个入口分别适合什么任务
- 能为自己的常见任务选对入口
- 理解组合使用的成熟路径

## 适合谁

已完成安装，想搞清楚不同入口差异的用户。

## 前置条件

完成第 2 课，已经跑通至少一个验证任务。

## 核心知识

### 一张平台决策表

| 你想做的事 | CLI | IDE 插件 | Desktop | Web | Cowork |
|----------|-----|---------|---------|-----|--------|
| 读懂一个代码仓库 | ✅ 首选 | ✅ 很顺手 | ✅ 可做 | ✅ 远程仓库适合 | ❌ |
| 改代码、修 Bug、跑测试 | ✅ 首选 | ✅ 首选 | ✅ 可做轻中度改动 | ⚠️ 更适合远程会话 | ❌ |
| 边看代码边对话 | ⚠️ | ✅ 最佳 | ✅ | ❌ | ❌ |
| 整理本地文档文件夹 | ⚠️ 可以但别扭 | ❌ | ✅ | ❌ | ✅ 首选 |
| 写报告、需求、总结 | ⚠️ | ❌ | ✅ | ✅ | ✅ 首选 |
| 浏览器辅助任务 | ⚠️ 通过 MCP | ⚠️ 通过 MCP | ✅ | ✅ | ✅ |
| 后台任务、计划任务 | ✅ 脚本化强 | ⚠️ | ✅ 本地调度 | ✅ Cloud tasks | ✅ Desktop 任务 |

### 五个入口的本质区别

- **CLI**：最适合开发者，控制力最强，脚本化最好
- **IDE**：最适合边改边看，特别适合 review、重构、逐文件迭代
- **Desktop**：Claude Code 的桌面形态，适合本地仓库、可视化 diff、桌面端调度
- **Web**：Claude Code 的云端形态，适合远程仓库、跨设备访问、长任务
- **Cowork**：Claude Desktop 里的任务模式，最适合非代码工作流，尤其是文档、资料、研究类任务

一句话区分：
- 想改代码、看 diff、跑命令：先看 CLI / IDE / Desktop / Web
- 想把“整理资料 -> 生成交付物”整段交给 Claude：先看 Cowork

### 两个典型错误

**错误一：把 Cowork 当成 Claude Code Web 版**——Cowork 不是 `claude.ai/code`。它运行在 Claude Desktop 里，是面向知识工作的任务模式。

**错误二：把仓库级重构放进纯文档入口**——大规模代码修改需要读目录、改文件、跑测试、看 diff、回退变更，在 CLI、IDE 或 Claude Code Desktop / Web 里更稳定。

### 一个更成熟的组合路径

Cowork 先整理需求文档、竞品资料、研究输入 → 沉淀到共享目录 → CLI / IDE / Desktop / Web 实现代码或脚本 → Cowork 或 Desktop 再生成面向业务的交付材料。

## 作业

回顾你上周最耗时的 3 个任务，给它们标注最合适的入口，不要只写"Claude"。

## 常见误区

1. **所有任务都用同一个入口**：不同任务适合不同入口
2. **把 Desktop、Web、Cowork 混成一个东西**：它们共享底层能力，但运行环境和任务边界不同
3. **把 Cowork 当成 CLI 的替代品**：它们适合完全不同的任务类型

## 验收标准

学员能为自己 3 个常见任务选对入口，并说出选择理由。
