## 核心知识

### 场景描述

你是一名市场研究分析师，每周需要处理大量文章、报告和数据，整理成简报发给管理层。以前完全手动，准备用 Cowork 建立可复用的"研究资料处理系统"。

### 文件结构设计

```
市场研究/
├── 原始资料/
│   ├── 2026-W14/     ← 本周收集
│   └── 存档/         ← 历史资料
├── 简报/
│   ├── 周报/
│   └── 专题/
├── 模板/
│   ├── 周报模板.md
│   └── 专题报告模板.md
└── 知识库/           ← 提炼出的长期洞察
```

四个目录各有分工：
- **原始资料**：按周归档的输入源，只增不改
- **简报**：周报和专题报告的输出目录
- **模板**：保证输出格式一致的模板文件
- **知识库**：从周报中提炼的长期有价值的洞察

### Global Instructions 配置

```
研究分析的输出要求：
- 先事实，再分析，再判断
- 数据必须注明来源（文件名或网址）
- 不确定的内容用「待核实」标注，不要猜测
- 结论控制在 3-5 条，不要堆砌观点
```

这四条规则确保了输出质量的底线：有据可查、不瞎猜、不啰嗦。

### 四阶段周工作流

**第一阶段：资料入库（每天随手做）**

把收集到的文章、截图放进 `原始资料/2026-W14/`。

这一步不需要 AI，关键是养成"随手归档"的习惯，确保周五处理时资料齐全。

**第二阶段：批量摘要（周五上午）**

```
读取"原始资料/2026-W14"里的所有文件，
为每份资料生成 3-5 句话摘要：
- 核心观点
- 关键数据
- 来源信息
输出到"原始资料/2026-W14/摘要汇总.md"
```

**第三阶段：生成周报**

```
基于"摘要汇总.md"，参照"模板/周报模板.md"，
生成本周市场简报：
- 本周重要动态（3-5 条）
- 值得关注的趋势（2-3 条）
- 建议管理层关注的事项（1-2 条）
保存到"简报/周报/2026-W14-市场简报.md"
```

**第四阶段：知识沉淀**

```
读取本周简报，判断哪些洞察有长期价值，
提炼后追加到"知识库/行业洞察.md"
```

### 三个月后的效果

建立这个系统三个月后，你会发现：
- **知识库积累**：大量提炼过的行业洞察，形成可检索的知识资产
- **质量稳定**：每周周报的质量越来越稳定，因为有了历史参照和固定模板
- **可传承**：新员工接手时，系统本身就是最好的培训材料——文件结构、模板、知识库一目了然

## Demo

用你自己的研究领域，搭建一个最小可用的资料处理系统：

1. 创建完整的文件结构（四个目录）
2. 编写 Global Instructions
3. 准备 3-5 份本周收集的资料放入原始资料目录
4. 走通四阶段工作流，生成一份周报
5. 从周报中提炼 1-2 条长期洞察写入知识库

## 作业

1. 按照课程中的结构创建你自己的"研究资料处理"Project，配置好 Global Instructions
2. 创建周报模板和专题报告模板，放入"模板"目录
3. 收集本周至少 5 份研究资料，放入对应周目录
4. 完整走通四阶段工作流：资料入库→批量摘要→生成周报→知识沉淀
5. 检查输出是否符合 Global Instructions 的四条要求（先事实再分析、注明来源、待核实标注、结论 3-5 条）
6. 连续执行 2-3 周，观察知识库的积累效果

## 常见误区

- **误区 1：不做文件结构直接开始**。没有清晰的目录分工，资料和输出混在一起，几周后就乱了。
- **误区 2：让 AI 猜测数据来源**。Global Instructions 里明确要求注明来源，如果 AI 无法确认来源，必须标注"待核实"。
- **误区 3：跳过摘要直接生成周报**。批量摘要是信息压缩的关键步骤，跳过它会导致周报质量不稳定。
- **误区 4：不做知识沉淀**。周报是短期输出，知识库才是长期资产。每周花 5 分钟提炼洞察，三个月后价值巨大。
- **误区 5：模板一成不变**。模板应该随着使用经验迭代优化，前几周的模板和三个月后的模板应该有明显进化。
