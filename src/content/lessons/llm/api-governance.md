## 这节课你要拿到什么

- 一套 Key 分离和命名规则（直接照抄就能用）
- 一张调用记录表（飞书多维表格模板）
- 一组预算分级规则

> 上节课你画好了模型地图，这节课解决接入后的管理问题——Key 怎么管、成本怎么控、记录怎么留。

## 第一步：把 Key 分开（⏱ 10 分钟）

不管你是个人还是小团队，只要开始接 API，第一件事就是把 Key 分开。

### 最少分 3 个 Key

| Key 名称 | 用途 | 额度建议 |
|----------|------|---------|
| test-[你的名字] | 开发测试、试新模型、调提示词 | 每月 10-20 元 |
| daily-[你的名字] | 日常工作流、正式任务 | 每月 50-100 元 |
| auto-[你的名字] | 自动化流程、批量任务 | 按任务量设，有上限 |

### 怎么操作

以 OpenAI 为例：
1. 登录 platform.openai.com，进入 API Keys 页面
2. 点击 Create new secret key
3. 命名为 test-[你的名字]，设置额度上限
4. 重复创建 daily 和 auto 两个 Key

DeepSeek、Anthropic 的操作类似，都在各自的开发者后台。

### Key 存放规则

- 存在密码管理器里（比如 1Password、Bitwarden），不要存在微信聊天记录或备忘录
- 不要写进公开代码仓库、截图或教程素材
- 如果用 .env 文件，确保 .gitignore 里有 .env

✅ 完成标准：你有 3 个分开的 Key，每个都有明确的用途和额度上限。

## 第二步：建一张调用记录表（⏱ 10 分钟）

不需要一开始上复杂平台，用飞书多维表格就够。

### 在飞书多维表格里建表

新建一个多维表格，命名为"API 调用记录"，加这些字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 调用日期 |
| 任务 | 单选 | 摘要/写作/提取/分析/批量 |
| 模型 | 单选 | 你用的模型名 |
| Key 类型 | 单选 | test/daily/auto |
| 输入量 | 数字 | 大约多少 token（粗估即可） |
| 输出量 | 数字 | 大约多少 token |
| 耗时 | 数字 | 秒 |
| 成本 | 数字 | 元或美元 |
| 结果可用 | 单选 | 是/否/需返工 |
| 备注 | 文本 | 特殊情况记录 |

### 示例数据

| 日期 | 任务 | 模型 | Key | 输入量 | 输出量 | 耗时 | 成本 | 结果可用 |
|------|------|------|-----|--------|--------|------|------|---------|
| 3/25 | 周报摘要 | GPT-5.4 mini | daily | 2000 | 800 | 15s | 0.01元 | 是 |
| 3/25 | 竞品分析 | Claude Sonnet 4 | daily | 8000 | 3000 | 45s | 0.15元 | 需返工 |
| 3/26 | 批量分类 | DeepSeek V3.2 | auto | 50000 | 5000 | 30s | 0.02元 | 是 |

只要你手工记录 10-20 次，就能发现哪类任务最烧钱、哪个模型返工最多、哪些调用其实不值得。

✅ 完成标准：你有一张调用记录表，至少记了 3 条真实数据。

## 第三步：给不同任务设预算档（⏱ 5 分钟）

按任务设预算，这样选模型就有边界：

| 预算档 | 任务类型 | 模型建议 | 单次成本上限 |
|--------|---------|---------|-------------|
| 低预算 | 批量分类、预处理、简单摘要 | DeepSeek V3.2 / GPT-5.4 nano | 0.05 元 |
| 中预算 | 内容提纲、方案框架、信息整理 | GPT-5.4 mini / Claude Sonnet 4 | 0.5 元 |
| 高预算 | 复杂分析、对外高质量内容、关键文档 | GPT-5.4 / Claude Opus 4.1 | 5 元 |

在飞书多维表格里加一个"预算档"字段，每次记录时标注。月底汇总看看各档花了多少。

✅ 完成标准：你的调用记录表里有预算档字段，知道每类任务的成本上限。

## 第四步：给高风险动作做日志（⏱ 5 分钟）

只要流程里涉及这些动作，就一定要留日志：

- 写文件（覆盖已有内容）
- 发消息（邮件、群通知）
- 调外部接口（支付、数据同步）
- 批量更新数据

日志至少记录：

```text
触发人：[你的名字]
执行时间：[日期时间]
模型：[用了哪个]
动作：[做了什么]
输入摘要：[大概内容]
输出摘要：[大概结果]
是否人工确认：[是/否]
```

个人用户可以直接在飞书文档里建一个"高风险操作日志"页面，每次手动记一条。等以后用 Coze 或其他工具做自动化时，再把日志接进去。

✅ 完成标准：你知道哪些动作需要留日志，有一个记录的地方。

## 个人用户的 4 条最小治理规则

直接照抄：

1. 所有实验调用都先走 test Key，不直接碰正式流程
2. 新流程连续跑满 5 次稳定后，再接入 daily Key
3. 任何涉及外发、写入、支付、批量修改的动作都必须人工确认
4. 每周五花 10 分钟回看调用记录，删掉无效调用、检查成本

## 案例：小张第一次接 API 的踩坑经历

小张是产品经理，想用 API 自动生成竞品周报。他直接在 Coze 里接了 OpenAI 的 API。

### 第 1 周：没做治理

| 问题 | 发生了什么 |
|------|-----------|
| Key 混用 | 测试和正式用同一个 Key，测试时把额度烧完了，正式流程断了 |
| 没有记录 | 不知道哪次调用花了多少钱，月底账单吓一跳 |
| 没有预算 | 调了一次 Claude Opus 4.1 做简单摘要，花了 2 块钱做了 2 毛钱的事 |

### 第 2 周：按这节课的方法补治理

| 改进 | 做了什么 |
|------|---------|
| Key 分离 | 建了 test/daily/auto 三个 Key |
| 调用记录 | 飞书多维表格记录每次调用 |
| 预算分级 | 竞品摘要用 DeepSeek，只有深度分析才用 Claude |
| 周回顾 | 每周五看一次记录，发现 30% 的调用其实不需要 |

> "第一周花了 80 块，第二周做了同样的事只花了 15 块。不是省钱，是知道钱花在哪了。"

## 常见翻车

### 翻车 1：Key 写进公开仓库

❌ 错误做法：把 API Key 直接写在代码里，推到 GitHub 公开仓库。

为什么错：爬虫会在几分钟内扫到你的 Key，然后用你的额度跑任务。有人因此一夜被刷掉几百美元。

✅ 正确做法：Key 存在 .env 文件里，.gitignore 加上 .env。或者用密码管理器存储。

### 翻车 2：所有任务都用最贵的模型

❌ 错误做法：觉得"贵的肯定好"，所有任务都用 Claude Opus 4.1 或 GPT-5.4。

为什么错：简单摘要用高端模型，就像用跑车送外卖——能送到，但完全没必要。

✅ 正确做法：按预算档分配模型。低预算任务用 DeepSeek V3.2 或 GPT-5.4 nano，省下来的钱留给真正需要高质量的任务。

### 翻车 3：从不回看调用记录

❌ 错误做法：接了 API 之后就不管了，月底看账单才发现超支。

为什么错：很多无效调用（测试忘关、重复调用、参数写错）会悄悄烧钱，不看记录根本发现不了。

✅ 正确做法：每周五花 10 分钟看一次调用记录，删掉无效调用，检查是否有异常。
