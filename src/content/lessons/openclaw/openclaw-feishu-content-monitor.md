## 这节课你要拿到什么

- 飞书内容监控台的完整配置路径
- Heartbeat 巡检规则配置
- 从"抓消息"到"产素材资产"的飞书多维表格结构

## 核心不是抓得多，而是筛得准

真正有价值的监控系统，能把线索分成三层：
- 异常变化 -> 立即推送
- 正常更新 -> 摘要
- 噪音 -> 静默

## 配置步骤

### 1. 安装技能

```bash
clawhub install web-search browser
openclaw skills list  # 确认安装成功
```

### 2. 配置 Heartbeat 巡检

```json
{
  "heartbeat": {
    "interval": 3600,
    "prompt": "执行内容巡检。检查以下来源：\n1. 搜索关键词：[关键词1]、[关键词2]\n2. 竞品动态：[竞品1]、[竞品2]\n\n按规则处理：\n- 异常增长或高频命中 -> 立即发飞书提醒\n- 常规更新 -> 写入今日素材摘要\n- 低相关内容 -> 静默记录"
  }
}
```

### 3. 配置 Cron 汇总

```json
{
  "cron": [
    {
      "pattern": "0 18 * * 1-5",
      "prompt": "汇总今天的内容监控结果，按优先级排序，发送到飞书。有价值的线索写入素材池文档。",
      "timezone": "Asia/Shanghai"
    }
  ]
}
```

### 4. 在 AGENTS.md 里写监控规则

```markdown
## 内容监控规则

### 立即提醒
- 竞品发布重大更新
- 关键词搜索量异常增长（比平时多 3 倍以上）
- 负面舆情或用户集中投诉

### 摘要
- 常规行业资讯
- 竞品日常内容发布

### 静默
- 广告和推广内容
- 重复抓取的旧内容

### 去重
- 同一条内容 24 小时内只提醒一次
```

### 5. 建飞书素材池（多维表格）

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| 标题 | 文本 | 线索的最短描述 |
| 来源 | 单选 | 来自哪个平台或竞品 |
| 命中关键词 | 文本 | 为什么被抓到 |
| 优先级 | 单选 | 立即 / 摘要 / 静默 |
| 建议动作 | 单选 | 观察 / 写稿 / 跟进 / 忽略 |
| 负责人 | 人员 | 谁来处理 |
| 状态 | 单选 | 待看 / 已入池 / 已转选题 / 已忽略 |

监控系统的终点不是群消息，而是这张表。

### 6. 手动测试

```bash
openclaw run --prompt "执行一次内容巡检测试。搜索关键词 [你的关键词]，把结果按立即/摘要/静默分类，发送到飞书。"
```

## 常见问题

| 问题 | 修法 |
| --- | --- |
| 太吵 | 缩窄关键词，把"立即"降级为"摘要" |
| 长期没新线索 | 扩来源，但一次只加 1 个 |
| 重复命中 | 加"同一内容 24 小时内只提醒一次" |

## 本课落地动作

- 配好 Heartbeat 和 Cron 规则
- 手动触发 1 次监控测试
- 把至少 3 条线索写进飞书素材池
