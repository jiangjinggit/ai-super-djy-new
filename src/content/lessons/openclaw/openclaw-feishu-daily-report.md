## 这节课你要拿到什么

- 一条从信息源到飞书推送的完整闭环（5 步跑通）
- 无翻墙 / 有翻墙两套信息源方案
- 可直接写进 AGENTS.md 的过滤规则模板
- 日报过空、过长、重复时的排错顺序

## 为什么从 AI 热点日报开始

这是 OpenClaw 最适合新手的第一个实战场景，因为它同时满足 4 个条件：

| 条件 | AI 热点日报为什么满足 |
| --- | --- |
| 输入源固定 | 36氪 AI 频道、GitHub Trending、少数派，不需要动态发现 |
| 价值直观 | 每天早上一条推送，开工就能用 |
| 风险可控 | 只读不写，不涉及删除、外发等敏感操作 |
| 容易收敛噪音 | 过滤规则可以持续迭代，错了也不会造成损失 |

## 5 步跑通最小闭环

前提：你已经完成了环境搭建（服务器 + 模型 + 飞书渠道），如果没有，先回上一课。

### 第 1 步：安装 web-search 技能并配好 Brave Search（3 分钟）

AI 热点日报需要从互联网抓取信息。OpenClaw 的搜索能力依赖 Brave Search API，不配 API Key 搜索基本不可用。

先申请 API Key（每月有 $5 免费额度，个人日报场景绰绰有余）：

1. 打开 [Brave Search API](https://brave.com/search/api/) 注册账号
2. 获取 API Key，$5 免费额度会自动到账
3. 无需绑卡即可开始使用

然后配置 OpenClaw：

```bash
# 交互式配置（推荐，会自动引导你填 Key）
openclaw configure --section web

# 或者手动写配置文件 ~/.openclaw/openclaw.json
# "tools": { "web": { "search": { "provider": "brave", "apiKey": "你的KEY" } } }
```

安装 web-search 技能：

```bash
openclaw skill install web-search
```

验证搜索能力是否正常：

```bash
openclaw skill list
# 输出里应该能看到 web-search
```

然后在聊天里发一句"搜索今天的 AI 新闻"，如果能返回带链接的结果，说明搜索配好了。

> 💡 Brave Search API 每月自动到账 $5 免费额度（Search 接口约 $0.003/次，$5 约等于 1600+ 次搜索）。日报每天跑一次、每次搜 4 个源，一个月大约 120 次，远在免费额度内。如果同时跑多个监控任务，注意控制频率。

### 第 2 步：配信息源（3 分钟）

把信息源写进 AGENTS.md，告诉 OpenClaw 去哪里抓内容。

**无翻墙方案（推荐先用这套跑通）：**

```markdown
## AI 热点日报 — 信息源
- 36氪 AI 频道（https://36kr.com/information/AI/）
- GitHub Trending 今日热门（https://github.com/trending）
- 少数派 AI 标签（https://sspai.com/tag/AI）
- 虎嗅前沿科技频道（https://www.huxiu.com/channel/105.html）
```

> 说明：OpenClaw 的 web-search 技能本质上是调用搜索引擎，不是直接爬取网页。上面的 URL 是告诉 OpenClaw 信息源的范围，实际抓取时它会通过搜索引擎获取这些站点的最新内容。如果某个源长期抓不到内容，换成搜索关键词效果更好，比如"site:36kr.com AI"。

**有翻墙方案（跑通后按需扩展）：**

在上面基础上加：

```markdown
- Hacker News 首页（https://news.ycombinator.com/）
- Product Hunt 今日热门（https://www.producthunt.com/）
- X/Twitter 关键账号：@sama、@AnthropicAI、@OpenAI
```

先用无翻墙版跑通再扩展，不要一上来就配太多源。

### 第 3 步：写过滤规则到 AGENTS.md（5 分钟）

信息源配好后，最关键的是告诉 OpenClaw 什么该留、什么该扔。没有过滤规则，日报要么太空要么太长。

```markdown
## AI 热点日报 — 过滤规则

### 保留条件（命中任一即保留）
- 涉及大模型 / AI Agent / AI 工具的新产品发布或重大更新
- GitHub 项目单日 star 增长超 500
- AI 领域融资超千万美元
- 主流 AI 公司（OpenAI / Anthropic / Google / Meta）的官方公告

### 过滤条件（命中任一即丢弃）
- 纯广告软文、品牌合作推广
- 昨日已推送过的同一事件（按标题相似度去重）
- 与 AI 无关的泛科技新闻

### 输出格式
- 每日最多 5 条，按重要性排序
- 每条格式：一句话摘要 + 来源链接
- 开头加一句"今日关键词"总结当天主题
```

### 第 4 步：配 Cron 定时触发（2 分钟）

让 OpenClaw 每个工作日早上 08:30 自动执行：

```json
{
  "cron": [
    {
      "pattern": "30 8 * * 1-5",
      "prompt": "执行 AI 热点日报任务：按 AGENTS.md 中的信息源和过滤规则，抓取今日 AI 领域热点，整理后推送到飞书。",
      "timezone": "Asia/Shanghai"
    }
  ]
}
```

注意：`timezone` 必须设成 `Asia/Shanghai`，否则 08:30 会按 UTC 算，变成下午 16:30。

### 第 5 步：手动触发验证（2 分钟）

不要等明天早上，先手动跑一次看效果：

```bash
openclaw run --prompt "执行 AI 热点日报任务：按 AGENTS.md 中的信息源和过滤规则，抓取今日 AI 领域热点，整理后推送到飞书。"
```

验证清单：

| 检查项 | 预期 | 不符合怎么办 |
| --- | --- | --- |
| 飞书收到消息 | 收到一条结构化日报 | 检查渠道连接状态 `openclaw gateway status` |
| 条数合理 | 3-5 条 | 太多 → 收紧过滤规则；太少 → 放宽保留条件 |
| 内容相关 | 都是 AI 相关 | 在过滤规则里加"与 AI 无关的一律丢弃" |
| 没有重复 | 同一事件只出现一次 | 加去重规则 |
| 有来源链接 | 每条带原文链接 | 在输出格式里明确要求 |

## 完整 AGENTS.md 配置参考

把上面的信息源、过滤规则、输出格式合在一起，就是一份完整的日报配置：

```markdown
## AI 热点日报

### 任务说明
每个工作日 08:30 自动执行，抓取 AI 领域热点，过滤整理后推送到飞书。

### 信息源
- 36氪 AI 频道（或搜索 site:36kr.com AI）
- GitHub Trending 今日热门
- 少数派 AI 标签
- 虎嗅前沿科技频道

### 保留条件（命中任一即保留）
- 大模型 / AI Agent / AI 工具的新产品或重大更新
- GitHub 单日 star 增长超 500
- AI 融资超千万美元
- OpenAI / Anthropic / Google / Meta 官方公告

### 过滤条件（命中任一即丢弃）
- 纯广告软文
- 昨日已推送内容（标题相似度去重）
- 与 AI 无关的泛科技新闻

### 输出格式
- 最多 5 条，按重要性排序
- 每条：一句话摘要 + 来源链接
- 开头加"今日关键词"

### 推送渠道
- 常规推送 → 飞书
- 重大突发（如 GPT-6 发布级别）→ 飞书 + Telegram
```

## 跑通之后：3 天观察期

跑通不等于配好。连续观察 3 天，按这个顺序调优：

| 天数 | 重点观察 | 调什么 |
| --- | --- | --- |
| 第 1 天 | 能不能收到、格式对不对 | Cron 时间、输出格式 |
| 第 2 天 | 内容质量、是否有噪音 | 过滤规则的保留/丢弃条件 |
| 第 3 天 | 是否有重复、信息源覆盖够不够 | 去重规则、信息源增减 |

## 最常见的 3 个故障

| 问题 | 常见原因 | 排查顺序 |
| --- | --- | --- |
| 日报太空（0-1 条） | ① Brave Search API Key 没配或过期 ② web-search 技能没装好 ③ 过滤规则太严 | 先发"搜索今天AI新闻"测搜索能力 → `openclaw skill list` 确认技能 → 放宽保留条件 |
| 日报太长（10+ 条） | ① 没设最大条数 ② 过滤规则太宽 ③ 保留条件太泛 | 先加"最多 5 条"限制 → 收紧过滤条件 → 提高保留门槛 |
| 信息重复 | ① 多个源报道同一事件 ② 没有去重规则 | 先加"按标题相似度去重" → 给信息源设优先级（同一事件只保留最早/最权威来源） |

## ✅ 本课落地动作

- 跑通 1 条飞书 AI 热点日报（手动触发验证成功）
- 连续观察 3 天，至少调整 1 次过滤规则
- 删掉 1 个低价值信息源或收紧 1 条过滤条件
