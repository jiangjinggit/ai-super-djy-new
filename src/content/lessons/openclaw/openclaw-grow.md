## 这节课你要拿到什么

- 系统"变笨"时的分层排查方法
- 周检和月检各看什么
- 怎么把稳定场景升级成资产

## 真正难的不是第一周，是一个月以后

系统长期稳定靠 3 套回路：

| 回路 | 管什么 |
| --- | --- |
| 行为回路 | SOUL.md、AGENTS.md |
| 信息回路 | USER.md、daily notes、MEMORY.md |
| 能力回路 | 技能、Nodes、主动任务 |

任何一个回路长期不回顾，系统都会开始跑偏。

## 出问题先分层查

| 症状 | 先查哪层 | 怎么修 |
| --- | --- | --- |
| 说话风格变了 | SOUL.md / AGENTS.md | 看规则是不是写宽了或互相冲突 |
| 不记得背景 | USER.md / MEMORY.md | 删过期信息，分清短期和长期 |
| 技能老报错 | 技能配置 / 授权 | 先查权限和 description |
| 通知太多或太少 | Heartbeat / Cron | 看条件是否过宽、阈值是否写清 |

## 周检和月检

| 频率 | 看什么 | 目标 |
| --- | --- | --- |
| 每周 | 通知噪音、失败任务、低价值规则 | 删掉明显无效的配置 |
| 每月 | 工作区结构、共享能力、节点权限、成本 | 判断该拆分、合并还是隔离 |

长期调优的目标不是功能越来越多，而是有效场景越来越稳、无效配置越来越少。

## 稳定两周以上的场景，该升级成资产

1. 写成场景 SOP（触发条件、输入源、输出格式）
2. 提炼成模板和规则资产
3. 对照场景库和案例模块，看能不能复制到第二个场景

## 本课落地动作

- 做一次分层排查
- 找出 1-3 条该删的低价值配置
- 写 1 份场景复盘
