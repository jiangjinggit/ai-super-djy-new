## 这节课你要拿到什么

- 三层记忆结构的分工
- "该不该进长期记忆"的判断方法
- 防止记忆污染的规则

## 记忆分 3 层

| 层级 | 放什么 | 例子 |
| --- | --- | --- |
| daily notes | 当天状态、巡检结果、临时反馈 | "今天客户 A 反馈了延迟问题" |
| MEMORY.md | 跨天仍有效的偏好和规律 | "我习惯先看结论再看细节" |
| 场景资产 | 已验证的 SOP、模板、失败案例 | "内容监控的标准巡检流程" |

## 系统变笨，通常不是记不住，而是记太多

这些东西别往 MEMORY.md 里塞：

| 不该进长期记忆 | 放哪里 |
| --- | --- |
| 当天情绪、一次性任务、临时优先级 | daily notes |
| 已结束的项目、失效的偏好 | 直接删掉 |
| 稳定可复用的流程 | 升级成场景资产 |

## 判断标准：7 天后它还成立吗？

- 不成立 -> daily notes
- 成立，且会反复影响行为 -> MEMORY.md
- 已经形成可复用流程 -> 升级成场景资产

## 升级路径

1. 当天巡检结果先写 daily notes
2. 连续多天都成立的规律再进 MEMORY.md
3. 稳定可复用的流程升级成场景资产

很多人第 2 步和第 3 步没分开，把流程写进长期记忆，MEMORY.md 越来越像垃圾场。

## 本课落地动作

- 清理 3 条不该在 MEMORY.md 里的信息
- 补 3 条真正该沉淀的长期偏好
- 从一个稳定流程里整理出 1 个场景资产
