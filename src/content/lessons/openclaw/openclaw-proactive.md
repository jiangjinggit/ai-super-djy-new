## 这节课你要拿到什么

- Heartbeat / Cron / 通知渠道的可执行配置
- 通知三层分流规则模板
- 从配置到验证的操作路径

## OpenClaw 的差异化在运行层，不在对话层

最值得学的不是它怎么聊天，而是它怎么自己开始工作。

| 部件 | 配置方式 | 适合 | 例子 |
| --- | --- | --- | --- |
| Heartbeat | `heartbeat.interval` + `heartbeat.prompt` | 高频检查、异常发现 | 每小时检查紧急邮件 |
| Cron | `cron[].pattern` + `cron[].prompt` | 固定时点汇总 | 每天 08:30 早报 |
| 通知渠道 | `openclaw channels` | 结果送达 | 飞书 + Telegram |

## Heartbeat 配置

```json
{
  "heartbeat": {
    "interval": 3600,
    "prompt": "检查是否有需要我注意的事项。查看邮箱是否有紧急邮件，检查日历是否有即将开始的会议冲突。有紧急事项立即通知，没有则不打扰。"
  }
}
```

`interval` 单位秒。3600 = 每小时，1800 = 每 30 分钟。

## Cron 配置

```json
{
  "cron": [
    {"pattern": "30 8 * * 1-5", "prompt": "生成今日工作日报", "timezone": "Asia/Shanghai"},
    {"pattern": "0 18 * * 1-5", "prompt": "生成今日工作总结", "timezone": "Asia/Shanghai"},
    {"pattern": "0 9 * * 1", "prompt": "生成本周周报", "timezone": "Asia/Shanghai"}
  ]
}
```

常用 cron 表达式：

| 表达式 | 含义 |
| --- | --- |
| `30 8 * * 1-5` | 工作日 08:30 |
| `0 18 * * 1-5` | 工作日 18:00 |
| `0 9 * * 1` | 每周一 09:00 |
| `*/30 9-18 * * 1-5` | 工作日 9-18 点每 30 分钟 |

注意：一定要设 `timezone: "Asia/Shanghai"`，否则时间按 UTC 算，08:30 会变成 16:30。

## 通知三层分流（写进 AGENTS.md）

```markdown
### 立即推送
- 日历冲突且 30 分钟内开始
- 来自 VIP 联系人的消息
- 命中关键词：紧急/故障/投诉/截止
-> 发飞书消息，必要时补 Telegram

### 合并摘要
- 普通邮件更新
- 常规内容监控命中
- 非紧急待办提醒
-> 并入早报或晚报

### 静默记录
- 广告和推广内容
- 重复性已知问题
- 低优先级订阅更新
-> 只写 daily notes，不主动打扰
```

## 最稳的起手顺序

| 阶段 | 做什么 | 不要做什么 |
| --- | --- | --- |
| 第 1 周 | 只上 1 个 Cron（早报） | 不要同时开多个主动任务 |
| 第 2 周 | 加 1 个 Heartbeat（邮件或日历检查） | 不要上自动外发 |
| 第 3 周 | 加内容监控或运营守夜 | 不要上高风险自动动作 |
| 第 4 周 | 回顾并删减低价值规则 | 不要继续加，先优化 |

## 刷屏排查

| 问题 | 修法 |
| --- | --- |
| 消息太多 | 缩窄 Heartbeat 的 prompt 条件 |
| 同一类消息重复发 | 在 AGENTS.md 加"同一事项 X 小时内不重复提醒" |
| 飞书群被噪音淹没 | 把常规信息从"立即"降级为"摘要" |
| 深夜被打扰 | 在 AGENTS.md 加静默时间规则 |

## 本课落地动作

- 配好 1 个 Cron 任务（早报）并手动触发测试
- 写好通知三层分流规则
- 连续观察 3 天是否过吵
