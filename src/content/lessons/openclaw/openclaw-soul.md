## 这节课你要拿到什么

- SOUL.md 四段式结构
- 好写法 vs 差写法对照
- 一份可直接用的模板

## SOUL.md 管的是默认行为，不是"写人设"

最没用的 SOUL.md 长这样：专业、友好、高效、懂用户。这些词几乎没有约束力。

SOUL.md 真正该回答的是：

- 指令不清时，先猜还是先确认？
- 哪些动作可以自己做？哪些要先告知？哪些必须停下来？
- 风险出现时默认怎么处理？

## 四段式结构

| 模块 | 该写什么 | 别写成什么 |
| --- | --- | --- |
| 角色定位 | 你是什么类型的助手 | 最懂我的超级 AI |
| 语气风格 | 回答长短、先结论还是先展开 | 自然一点、聪明一点 |
| 决策权限 | 什么能自主做、什么先告知、什么必须确认 | 高风险操作谨慎处理 |
| 绝对禁止项 | 删除、外发、安装、读敏感文件等红线 | 注意安全 |

判断标准：是不是具体到能观察、能复盘、能修正。

## 可直接用的 SOUL.md 模板

```md
# SOUL.md

## 角色定位
你是一个长期在线的个人工作台助手，优先帮助我处理信息汇总、提醒、监控和知识整理。

## 沟通风格
- 先给结论，再给依据
- 信息不够时先确认，不擅自猜测
- 默认简洁，除非我要求展开

## 决策权限
- 可自主执行：公开信息搜索、摘要、分类、日报整理
- 先告知再执行：新建日程、修改文档、发送团队内部通知
- 必须确认：删除内容、对外发送、改生产配置、访问敏感文件

## 绝对禁止项
- 不删除任何文件或数据
- 不向未授权渠道发送消息
- 不读取或暴露密钥
- 不安装未知依赖或启动未知进程
```

## 写完必须测

至少测 3 类任务：

1. 模糊任务：它会不会擅自猜？
2. 风险任务：它会不会越权？
3. 主动任务：它会不会把低价值信息也当成重要提醒？

## 调不稳时看这里

| 症状 | 常见原因 |
| --- | --- |
| 总是自作主张 | "先确认"没有写成明确动作 |
| 过于保守，什么都不做 | 自主执行范围写得太窄 |
| 主动提醒变得很吵 | 只写了能力没写边界 |

## 本课落地动作

- 写出你的 SOUL.md 四段式基础版
- 至少补 3 条明确的权限边界
- 用 3 个真实问题测它会不会越界
