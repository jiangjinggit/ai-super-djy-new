## 这节课你要拿到什么

- 一套不会越记越乱的 AI 知识库结构
- 一个“成功案例 + 失败复盘”双轨记录方法
- 一份每周 20 分钟的复盘流程

## 先定规则：只存以后还会再用的东西

不要把所有对话截图都丢进知识库。真正该存的只有 4 类：

- 可复用模板
- 已验证有效的流程
- 典型案例
- 失败复盘

不建议存：

- 一次性闲聊
- 没有后续价值的碎片资料
- 没有标注场景的 Prompt

## 推荐的知识库结构

你可以直接按这个目录来：

```text
01-模板
02-流程
03-案例
04-复盘
05-待验证
```

### 01-模板

放这些内容：

- 文章提纲模板
- 纪要模板
- 客服回复模板
- 资料摘要模板

### 02-流程

放这些内容：

- 内容生产 SOP
- 报价流程
- 信息整理流程

### 03-案例

每个案例至少写：

- 场景
- 输入
- 用了什么提示词
- 输出效果
- 适合复制到哪些任务

### 04-复盘

每次失败都不要只写“效果不好”，而要写清楚：

- 原始任务是什么
- 哪一步出了问题
- 是输入不清、提示词不清，还是任务本身不适合 AI
- 下次怎么改

## 每周 20 分钟复盘流程

每周固定一次，按顺序做：

1. 回看本周 3 个最好结果
2. 回看本周 3 个最差结果
3. 把有效模板移到“01-模板”
4. 把失败案例移到“04-复盘”
5. 删除无价值记录

## 一个很关键的动作：给内容打标签

建议至少打 4 个标签：

- 任务类型：摘要 / 写作 / 分析 / 客服 / 方案
- 场景：个人 / 团队 / 客户 / 内容 / 开发
- 输出类型：表格 / 列表 / 纪要 / 文案
- 结果质量：高 / 中 / 低

以后你找模板会非常快。

## 本周落地动作

今天就做：

1. 建一个 `AI 模板库` 文档或文件夹
2. 把你目前最常用的 3 个 Prompt 放进去
3. 新建一个 `失败复盘` 页面
4. 把最近一次失败任务按模板写下来

做到这一步，你的 AI 使用方式就开始从“临时问”升级为“系统积累”了。
