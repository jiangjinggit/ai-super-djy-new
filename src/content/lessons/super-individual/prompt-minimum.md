## 这节课你要拿到什么

- 一个稳定可复用的 Prompt 结构
- 一个从模糊需求改写成可执行输入的方法
- 三个可以直接套用的提示词模板

## Prompt 最小公式

不要追求“神级提示词”，先把下面 5 个元素写全：

1. `角色`：你希望 AI 站在什么身份做这件事
2. `目标`：要完成什么结果
3. `上下文`：背景、对象、素材、限制条件
4. `约束`：长度、语气、禁区、必须包含什么
5. `输出格式`：表格、分点、JSON、邮件稿、脚本

你会发现，大部分“AI 不好用”的问题，其实是这 5 项没说清楚。

## 错误示范和正确示范

### 错误示范

```text
帮我写个方案
```

问题：

- 没有对象
- 没有目标
- 没有输出标准
- 没有长度和结构要求

### 正确示范

```text
你是一个内容运营顾问。

请为一个刚起步的 AI 学习公众号，输出 3 个 7 天选题方案。

背景：
- 读者是 25-35 岁上班族
- 主题聚焦 AI 提效和副业尝试
- 目标是提高收藏率和私域加粉

约束：
- 每个方案都要包含主题、核心角度、适合的标题方向、推荐 CTA
- 不要写空泛趋势判断
- 内容要能直接执行

输出格式：
用 Markdown 表格输出
```

## 把模糊需求改写成结构化任务

以后你每次提问，先按下面 4 步自己改写一遍：

### 第 1 步：把“想法”改成“动作”

不要说“帮我想想”，改成：

- 帮我列出
- 帮我对比
- 帮我重写
- 帮我整理成表格
- 帮我输出执行计划

### 第 2 步：把“对象”说清楚

说明这份内容给谁用：

- 给老板看
- 给客户看
- 给粉丝看
- 给自己复盘

### 第 3 步：把“限制条件”写清楚

至少补 3 类限制：

- 时间限制：今天内交付、3 天内执行
- 风格限制：专业、克制、简洁、适合新手
- 内容限制：不要承诺收益、不要写无法验证的数据

### 第 4 步：指定输出格式

如果你不指定，AI 就会按它最舒服的方式输出，不一定是你最省事的方式。

优先用这些格式：

- 表格：做对比、做清单、做选型
- 列表：做步骤、做 SOP、做检查项
- JSON：给系统、自动化或开发接入
- 模板：给重复任务复用

## 三个高频模板

### 模板 1：写作提纲

```text
你是我的内容编辑。

请围绕主题：[主题]
面向对象：[读者]
目标：[想让读者获得什么]

输出：
1. 一个清晰的大纲
2. 每一部分的核心观点
3. 建议补充的数据或案例

约束：
- 不要写空话
- 每一部分都要能落到具体动作
- 语言适合中文互联网读者
```

### 模板 2：资料整理

```text
请把以下资料整理成执行摘要。

目标读者：[老板/客户/团队成员]
我最关心的维度：[成本/风险/时间/可行性]

输出格式：
1. 三句话总结
2. 关键要点列表
3. 风险和待确认项
4. 下一步建议

资料如下：
[粘贴原文]
```

### 模板 3：流程设计

```text
你是流程设计顾问。

请帮我把下面的重复任务整理成 SOP。

任务名称：[任务]
输入材料：[有哪些输入]
完成标准：[什么算完成]
当前问题：[最耗时、最容易错在哪里]

输出：
1. 流程步骤
2. 每一步所需输入
3. 哪一步可以交给 AI
4. 哪一步必须人工确认
5. 最终产出模板
```

## 跟着做一遍：把一句模糊话改成可用提示词

原始说法：

```text
帮我写一个公众号选题
```

把它一步步补全：

### 第 1 步：补角色

```text
你是我的内容编辑。
```

### 第 2 步：补目标

```text
请输出 5 个适合 AI 入门读者的公众号选题。
```

### 第 3 步：补上下文

```text
读者是 25-35 岁上班族，刚开始接触 AI，最关心提效和副业尝试。
```

### 第 4 步：补约束

```text
不要写空泛趋势题；每个选题都要有一个明确问题切口。
```

### 第 5 步：补输出格式

```text
用表格输出：选题、核心问题、适合标题方向。
```

最后得到的就是一个可以真正拿去跑的提示词。

## 新手执行规则

以后每次提问前，先检查：

- 这次是让 AI 写、比、拆、整理，还是判断
- 我有没有把对象说清楚
- 我有没有指定输出格式

这 3 件事做到了，大部分提示词已经能用了。

## 自检清单

每次发给 AI 前，至少确认：

- 角色是否明确
- 目标是否具体
- 上下文是否足够
- 约束是否写清楚
- 输出格式是否指定

如果缺两项以上，不要怪模型，先改输入。
