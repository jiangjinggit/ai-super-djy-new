## 这节课你要拿到什么

- 一个能从零做出来的最小工作流模板
- 一种把重复任务拆成可复用 SOP 的方法
- 一个适合新手的“半自动优先”落地顺序

## 第一步：找到你的第一个高频任务

不要一开始做复杂自动化，先找一个满足下面条件的任务：

- 一周至少做 3 次
- 输入材料相对固定
- 输出格式相对固定
- 错了也不会造成大损失

适合起手的任务：

- 会议纪要整理
- 文章提纲初稿
- FAQ 回答草稿
- 资料摘要
- 客户跟进记录整理

## 第二步：把任务拆成 5 个格子

用下面这个结构，先手写一版：

| 格子 | 你要写什么 |
| --- | --- |
| 任务目标 | 这个流程最终要产出什么 |
| 输入 | 原始资料从哪里来 |
| 处理步骤 | 按顺序要做哪些动作 |
| 输出 | 最终要交付什么格式 |
| 完成标准 | 什么样的结果算合格 |

### 示例：会议纪要流程

| 格子 | 内容 |
| --- | --- |
| 任务目标 | 把 1 小时会议录音整理成纪要和待办 |
| 输入 | 录音转写文本、会议主题、参会人 |
| 处理步骤 | 清理噪音 -> 提炼核心结论 -> 列待办 -> 标负责人 |
| 输出 | 一份 Markdown 纪要 |
| 完成标准 | 有结论、有待办、有负责人、有截止时间 |

## 第三步：给每一步标记“AI 做”还是“人做”

最小工作流不要全自动，先分成 3 类：

- `AI 做`：摘要、分类、初稿、格式整理
- `AI 做，人确认`：对外话术、分析结论、方案建议
- `必须人做`：最终拍板、外发、敏感信息处理

### 示例

1. 粘贴会议转写文本：人做
2. 清理口语和重复句：AI 做
3. 提炼决策点：AI 做
4. 确认负责人和截止时间：人做
5. 生成最终纪要格式：AI 做
6. 发到群里：人确认后发送

## 第四步：把变量固定下来

一个流程能不能复用，关键看变量有没有收拢。

先把每次都要变的内容单独列出来：

- 主题
- 对象
- 输入材料
- 风格
- 输出格式

然后做成固定输入卡片：

```text
任务名称：
目标对象：
原始材料：
输出格式：
必须包含：
不能出现：
```

以后只换变量，不重写整段需求。

## 可直接复制的工作流模板

```text
你是我的流程执行助手。请根据下面的固定模板完成任务。

任务名称：
[填写]

目标：
[填写]

输入材料：
[填写]

处理要求：
1. 先阅读全部材料
2. 按步骤处理
3. 输出时标记待确认项

输出格式：
[表格/列表/邮件/纪要]

必须包含：
[填写]

不能出现：
[填写]
```

## 第五步：先手动跑 3 次，再考虑自动化

不要直接接 API 或自动化平台，先手工跑 3 次，观察：

- 哪一步最慢
- 哪一步最容易返工
- 哪一步最依赖人工判断

如果 3 次之后流程还经常变，说明你还没把任务定义清楚，不适合自动化。

## 可直接照抄的会议纪要工作流

如果你完全没想法，先从这个流程开始：

### 输入

- 会议转写文本
- 会议主题
- 参会人名单

### 提示词

```text
你是会议纪要助手。

请根据以下会议转写文本，输出一份纪要。

要求：
1. 先给三句话总结
2. 再列出关键结论
3. 再列出待办事项、负责人和截止时间
4. 如果原文里没有负责人或时间，标记“待确认”

会议主题：
[填写]

参会人：
[填写]

会议转写：
[粘贴文本]
```

### 人工确认点

- 负责人是否准确
- 截止时间是否存在
- 结论是否漏项

先把这一个流程跑顺，你就已经有了第一个像样的工作流。

## 完成标准

这节课做完后，你至少应该产出：

- 1 个高频任务工作流
- 1 份固定输入模板
- 1 个 AI 执行提示词
- 1 套人工确认点

这 4 个东西比“学会很多概念”更值钱。

## 本课落地动作

1. 选一个高频任务，用 5 格拆解法写出工作流
2. 给每一步标记"AI 做"还是"人做"
3. 用对话助手（豆包/元宝/通义千问/Kimi）手动跑一次，记录哪里要改
